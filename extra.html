심화 자료 — 자동화(cron)

사람이 매일 손으로 하던 일을 컴퓨터가 정해진 시간에 대신 하게 만드는 방법입니다. 그 “정해진 시간”을 적는 문법이 cron(크론)이고, 이 페이지는 그 한 가지를 끝까지 다룹니다. 지금 다 읽지 않아도 됩니다. 필요해질 때 찾아 읽으면 됩니다.

💡 이 페이지에서 배우는 것
  • 왜 자동화가 필요한지 — 잠자는 내 앱 깨우기
  • cron 다섯 칸을 읽고 쓰는 법 (이 페이지의 핵심)
  • 한국시간을 UTC로 바꿔 적는 법과 자주 걸리는 함정
  • GitHub Actions에 얹어 실제로 돌리기

1. 왜 자동화? — 잠자는 내 앱 깨우기

  • 무료 스트림릿 앱의 성질 = 방문이 끊기면 잠듦 → 다음 방문자가 “깨우기” 화면에서 1~2분 대기
  • 해결 = 로봇에게 시간표를 주고 대신 찾아가게 하기
  • 그 시간표를 적는 문법이 cron → 아래에서 이것만 자세히 봅니다

2. cron 완전 정복

  • cron = “언제 실행할지”를 다섯 칸으로 적는 약속
  • 칸은 띄어쓰기로 구분하고, 순서가 정해져 있습니다
cron 다섯 칸의 순서
┌─────── 분 (0-59)
│ ┌───── 시 (0-23)
│ │ ┌─── 일 (1-31)
│ │ │ ┌─ 월 (1-12)
│ │ │ │ ┌ 요일 (0-6, 0이 일요일)
│ │ │ │ │
5 * * * *

위 5 * * * *를 왼쪽부터 읽으면 “5분에 · 매 시 · 매 일 · 매 월 · 매 요일” → 매시간 5분마다가 됩니다.

각 칸에 쓸 수 있는 값

칸범위메모
① 분0 ~ 59가장 자주 쓰는 칸
② 시0 ~ 2324시간제 · UTC 기준(아래 참고)
③ 일1 ~ 31매월 며칠
④ 월1 ~ 121월 ~ 12월
⑤ 요일0 ~ 60=일 1=월 2=화 3=수 4=목 5=금 6=토 · SUN~SAT 이름도 가능

기호 네 개면 끝

기호뜻예시
*모든 값 (매번)시 칸에 * = 매시간
,여러 개 열거시 칸에 0,12 = 0시와 12시
-범위요일 칸에 1-5 = 월요일부터 금요일
/간격분 칸에 */15 = 15분마다
✔️ 읽는 연습 — 소리 내어 왼쪽부터
  • 0 * * * * → “0분에, 매시간” = 매시 정각
  • */10 * * * * → “10분 간격으로” = 10분마다
  • 30 3 * * * → “3시 30분에, 매일” = 매일 03:30
  • 0 0 1 * * → “0시 0분, 1일에” = 매월 1일 자정
  • 0 9 * * 1 → “9시 정각, 월요일에” = 매주 월요일 09:00

가장 중요한 것 — 시간은 UTC입니다

  • GitHub Actions의 cron은 세계표준시(UTC)로 해석됩니다 (한국시간 아님)
  • 한국시간(KST) = UTC + 9시간
  • 그래서 cron에 적을 때는 원하는 한국시간에서 9를 뺍니다
⚠️ 9를 뺐더니 음수가 되면 → 전날로 넘어갑니다
  • 한국시간이 09시 이상이면 → 그냥 9를 뺀 값 (같은 날)
  • 한국시간이 09시 미만이면 → +15를 하고 전날로 (예: 07시 → 22시 전날)
  • 예) 한국 07:23 → UTC 22:23 · 한국 12:00 → UTC 03:00

실전 변환표 — 하고 싶은 것 → 이렇게 적기

원하는 시점 (한국시간)cron
매시간 5분마다5 * * * * (분만 정하면 시차와 무관)
15분마다*/15 * * * *
6시간마다0 */6 * * *
매일 아침 09:000 0 * * * (9-9=0)
매일 낮 12:000 3 * * * (12-9=3)
매일 저녁 19:2323 10 * * * (19-9=10)
매일 아침 07:2323 22 * * * (7+15=22 · 전날)
매주 월요일 09:000 0 * * 1
평일(월~금) 09:000 0 * * 1-5
매월 1일 09:000 0 1 * *
⚠️ 요일과 이른 아침을 같이 쓸 때 — 요일도 하루 밀립니다
  • “한국시간 월요일 아침 7시”를 원하면 → UTC로는 일요일 22시
  • 그래서 요일 칸에 1(월)이 아니라 0(일)을 적어야 합니다 → 0 22 * * 0
  • 시각이 한국시간 09시 미만이면 요일·날짜가 하루 앞으로 당겨진다고 기억하세요
💡 확인 도구와 AI 활용
  • crontab.guru — 다섯 칸을 넣으면 영어 문장으로 풀어서 보여 줍니다 (시차는 계산해 주지 않으니 UTC로 넣어 확인)
  • 헷갈리면 아래처럼 AI에게 물어보면 됩니다 — 강의 내내 한 방식 그대로
① 샘플 프롬프트 — cron 물어보기
깃허브 액션즈로 자동 실행 시간표를 만들려고 해.
한국시간으로 평일 아침 8시 30분에 실행되게 하고 싶어.

깃허브 액션즈의 시간표(cron)는 세계표준시(UTC) 기준이라고 들었어.

3. GitHub Actions에 얹어 실제로 돌리기

  • 저장소에 .github/workflows/ 폴더를 만들고 .yml 파일 하나를 둡니다
  • 그 안에 cron과 실행할 명령을 적으면 끝
② 개념 예시 (.github/workflows/collect.yml)
name: 데이터 수집
on:
  schedule:
    - cron: "5 * * * *"      # 매시간 5분에 실행 (UTC 기준!)
  workflow_dispatch:          # 이 줄이 있으면 '수동 실행' 버튼도 생김
jobs:
  run:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: python collect.py
        env:
          API_KEY: ${{ secrets.API_KEY }}   # 비밀키는 Secrets에서
💡 두 가지만 챙기면 됩니다
  • 비밀키 = 코드에 직접 쓰지 않기 → Settings → Secrets and variables → Actions에 넣고 ${{ secrets.API_KEY }}로 꺼내기
  • workflow_dispatch = Actions 탭에 수동 실행 버튼이 생김 → 시간표를 기다리지 않고 지금 바로 시험

4. 꼭 알아 둘 제약과 함정

⚠️ 다섯 가지
  • UTC 기준 — 한국시간 −9. 9시간 어긋나는 실수가 가장 흔합니다
  • 정시보다 늦게 돕니다 — 무료 러너가 붐비면 몇 분 지연이 정상. 매시 0분보다 5분처럼 살짝 뒤를 권합니다
  • 최소 간격 5분 — 1분마다 같은 설정은 그대로 동작하지 않습니다
  • 기본 브랜치에서만 시간표가 동작합니다
  • 60일간 저장소에 아무 활동이 없으면 스케줄이 자동으로 멈춥니다 → 가끔 커밋하거나 수동 실행 버튼을 눌러 깨워 두기
