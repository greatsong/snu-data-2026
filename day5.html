🎯 오늘 하는 것

0. 지난 차시 핵심 정리

  • 1일 GitHub Pages 배포 · 바이브코딩 ①②③ · 키 없는 API
  • 2일 데이터 분석·시각화(pandas·Plotly) · 지도 · 비판적 사고
  • 3일 API·키·secrets · 나만의 AI 채팅앱
  • 4일 Supabase(DB) 저장 · 머신러닝(예측·외삽 주의)
  • 오늘 = 조각을 모아 내 프로젝트 발표

1. 프로젝트 마무리 — 발표할 수 있게

  • 거창하게 고칠 필요 없음
  • 발표에서 자랑할 딱 세 가지만 또렷하게 ↓
발표에서 말할 것한 줄로 준비
무슨 질문예) “서울은 100년 동안 얼마나 더워졌나?”
무슨 데이터예) 기상청 기온, 우리 반 설문(Supabase에 쌓은 것)
무슨 발견예) “여름이 3주 길어졌다”
  1. 패들렛에 게시물 하나 올리기
    작품 게시판(패들렛)에 제목 = 내 질문 한 줄, 본문에 앱 주소 · 발견 한 줄 · 이름을 적어 올립니다. 오늘 발표는 이 게시판에서 합니다 — 아래 샘플처럼 올리면 됩니다.
  2. 발표 1장 요약 만들기
    질문·데이터·발견 세 줄 = 발표 대본. 화면엔 내 앱, 세 줄만 말하면 충분
✔️ 게시물 샘플 — 이렇게 올리면 됩니다
샘플 1 · 인구
“우리 동네, 젊어지고 있을까?”
앱: my-town-card.streamlit.app
발견: “신도시 동만 평균연령이 8살 어리다” — 김○○
샘플 2 · 기온
“2050년 서울 여름은 몇 도?”
앱: seoul-temp-2050.streamlit.app
발견: “직선만 그어도 100년에 +2℃” — 이○○
샘플 3 · 유튜브
“내 최애 채널, 언제 올릴 때 터질까?”
앱: my-channel-lab.streamlit.app
발견: “목요일 업로드가 조회수 1.6배” — 박○○
💡 완성도보다 이야기
  • 버튼 덜 예뻐도 OK
  • 주인공 = “무엇이 궁금해서, 무엇을 발견했나”
  • 미완성이어도 이야기 있으면 훌륭한 발표
🏘️ 지도를 만든 사람 → ‘우리 동네 카드’로 마무리
  • 2일차 우리 지역 지도를 중심에 두고 앱 한 화면으로
  • 지도 + 지표 카드 3개(총인구·평균연령·고령화율) + 동 고령화율 top 5 표
  • 여유 되면 연도 슬라이더(2015→2026)로 우리 동네 변화까지
  • 발표 세 줄: “우리 ○○구를 동 단위로 봤더니 __가 보였다”

2. 발표하는 법 — 갤러리 워크

  • 전원 발표 = 시간 부족 → 패들렛을 미술관처럼 둘러보듯 서로 관람
  1. 전원 URL 게시
    패들렛에 모두의 앱 주소 집결 = 오늘의 전시장
  2. 패들렛에서 서로의 앱 눌러 보기
    다른 앱 직접 눌러 보기 → 좋아요 + 한 줄 피드백(댓글/스티커)
  3. 하이라이트 발표(화면 공유)
    몇 명이 앞에서 2~3분 소개. 질문·데이터·발견 세 줄이면 끝
  4. 비판적 질문 1개
    예) “이 숫자, 다른 해랑 비교하면?” “이 데이터는 언제 걸까?” → 가장 값진 습관
✔️ ‘비판적 사고’가 왜 중요?
  • 그냥 믿지 않고 “정말일까? 어디서 왔을까?” 되묻기
  • = 데이터 분석의 진짜 실력
  • 서로 질문 던지기 → 내 프로젝트 보는 눈

  • 오늘 다 안 봐도 됩니다 · 강사 안내에 따라, 또는 집에서 천천히

🎒 오늘의 체크 & 마무리

  • 내 앱 URL을 패들렛에 게시했다
  • 발표 세 줄(질문·데이터·발견)을 준비했다
  • 다른 사람 앱을 체험하고 좋아요·피드백을 남겼다
  • 심화 자료(진짜 DB·자동화·API)가 어디 있는지 안다
  • 데이터에 비판적 질문을 던질 줄 안다
