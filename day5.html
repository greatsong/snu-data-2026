0. 지난 차시 핵심 정리

  • 1일 GitHub Pages 배포 · 바이브코딩 ①②③ · 키 없는 API
  • 2일 데이터 분석·시각화(pandas·Plotly) · 지도 · 비판적 사고

1. 프로젝트 마무리 — 발표할 수 있게

  • 거창하게 고칠 필요 없음
  • 발표에서 자랑할 딱 세 가지만 또렷하게 ↓
발표에서 말할 것한 줄로 준비
무슨 질문예) “서울은 100년 동안 얼마나 더워졌나?”
무슨 데이터예) 기상청 기온, 우리 반 설문(Supabase에 쌓은 것)
무슨 발견예) “여름이 3주 길어졌다”
  1. 갤러리에 작품 올리기
    작품 갤러리에서 작품 제출 탭을 열고 닉네임 · 앱 주소 · 한 줄 소개를 적어 올립니다. 오늘 발표는 이 갤러리에서 합니다. 게시 전에 내 앱 주소를 한 번 열어 확인하세요. 잠들어 있으면 지금 깨워 두어야 갤러리 워크 때 바로 열립니다.
  2. 발표 1장 요약 만들기
    질문·데이터·발견 세 줄 = 발표 대본. 화면엔 내 앱, 세 줄만 말하면 충분
✔️ 게시물 샘플 — 이렇게 올리면 됩니다
샘플 1 · 인구
“우리 동네, 젊어지고 있을까?”
앱: my-town-card.streamlit.app
발견: “신도시 동만 평균연령이 8살 어리다” — 김○○
샘플 2 · 기온
“2050년 서울 여름은 몇 도?”
앱: seoul-temp-2050.streamlit.app
발견: “직선만 그어도 100년에 +2℃” — 이○○
샘플 3 · 유튜브
“내 최애 채널, 언제 올릴 때 터질까?”
앱: my-channel-lab.streamlit.app
발견: “목요일 업로드가 조회수 1.6배” — 박○○
샘플 4 · 논문
“우리 분야 논문, 얼마나 쏟아지고 있나?”
앱: my-field-trend.streamlit.app
발견: “제목에 ‘LLM’이 든 논문, 3년 새 49편 → 17,752편” — 정○○
💡 완성도보다 이야기
  • 버튼 덜 예뻐도 OK
  • 주인공 = “무엇이 궁금해서, 무엇을 발견했나”
  • 미완성이어도 이야기 있으면 훌륭한 발표
🏘️ 지도를 만든 사람 → ‘우리 동네 카드’로 마무리
  • 2일차 우리 지역 지도를 중심에 두고 앱 한 화면으로
  • 지도 + 지표 카드 3개(총인구·평균연령·고령화율) + 동 고령화율 top 5 표
  • 여유 되면 연도 슬라이더(2015→2026)로 우리 동네 변화까지
  • 발표 세 줄: “우리 ○○구를 동 단위로 봤더니 __가 보였다”

2. 발표하는 법 — 갤러리 워크

  • 오늘 발표는 작품 갤러리에서 서로의 앱을 둘러보는 방식입니다
  • 진행 방법은 강사 안내에 따라 함께 합니다

3. 보안이 중요한 데이터라면 — 내 컴퓨터에서만 돌리기

  • 성적·상담 기록·개인정보·병원 자료처럼 남에게 보이면 안 되는 데이터가 있습니다
  • 지금까지 한 클라우드 배포는 데이터를 인터넷에 올리는 방식 → 이런 데이터엔 안 맞습니다
  • 대신 로컬 실행: 앱이 내 컴퓨터에서만 돌고, 데이터는 이 컴퓨터 밖으로 한 발도 안 나갑니다
  • 2일차 네 칸 표의 🔒 로컬 Streamlit 칸이 바로 이것입니다 (2일차 다시 보기)
이런 데이터라면어디서 실행?
성적·상담·개인정보·병원기록 등 공개하면 안 됨🔒 로컬 실행 (내 컴퓨터에서만)
공개해도 되는 데이터 · 링크로 서로 공유🌍 클라우드 배포 (지금까지 한 방식)
아직 다듬는 중 · 나만 확인🔒 로컬 실행
📁 먼저 준비 — 폴더 하나에 다 모으기
  • 3·4일차에 만든 앱 코드 파일은 main.py 이름 그대로 둡니다
  • 같은 폴더에 민감한 데이터 파일(csv·엑셀)을 둡니다 → 앱은 이 폴더 안에서만 읽습니다
  • 아래 안내는 main.py와 데이터 파일이 다운로드 폴더에 있다고 가정합니다 — 다른 곳에 뒀다면 3번 명령의 경로만 바꾸면 됩니다

실행은 세 걸음입니다. 처음 한 번만 파이썬·스트림릿을 깔고, 그 뒤로는 명령 한 줄이면 됩니다.

  1. 파이썬 설치 (처음 한 번)
    이미 있으면 넘어가도 됩니다. 없으면 python.org/downloads에서 내려받아 설치하세요. 윈도우는 설치 첫 화면의 Add Python to PATH를 꼭 체크해야 다음 명령이 먹힙니다.
  2. 터미널 열고 스트림릿 설치 (처음 한 번)
    맥은 터미널(⌘+Space → “터미널”), 윈도우는 명령 프롬프트(시작 → “cmd”)를 엽니다. 아래 명령을 한 줄씩 복사해 차례로 붙여넣고 Enter 하세요.
  3. 앱 폴더에서 실행
    streamlit run main.py를 실행하면 브라우저가 저절로 열리고 localhost:8501 주소에 내 앱이 뜹니다. 이 주소는 내 컴퓨터에서만 열립니다.

🍎 맥 — 터미널에 한 줄씩

1️⃣ 파이썬 확인 — 3.x 가 보이면 성공
python3 --version
2️⃣ 스트림릿 설치 — 처음 한 번만
pip3 install streamlit
3️⃣ 다운로드 폴더로 이동
cd ~/Downloads
4️⃣ 실행 — 브라우저가 저절로 열립니다
streamlit run main.py
💡 파일을 다운로드 폴더가 아닌 곳에 뒀다면
  • 터미널에 cd 를 치고 한 칸 띄운 뒤, 그 폴더를 터미널 창으로 끌어다 놓으면 경로가 저절로 붙습니다 → Enter

🪟 윈도우 — 명령 프롬프트에 한 줄씩

1️⃣ 파이썬 확인 — 3.x 가 보이면 성공
python --version
2️⃣ 스트림릿 설치 — 처음 한 번만
pip install streamlit
3️⃣ 다운로드 폴더로 이동
cd %USERPROFILE%\Downloads
4️⃣ 실행 — 브라우저가 저절로 열립니다
streamlit run main.py
💡 파일을 다운로드 폴더가 아닌 곳에 뒀다면
  • 탐색기에서 그 폴더를 연 뒤 주소창에 cmd 를 치고 Enter 하면, 그 폴더에서 바로 명령 프롬프트가 열립니다 → 3번은 건너뛰고 4번만 실행
💡 잘 깔렸는지 먼저 시험 — streamlit hello
  • 내 앱을 돌리기 전에 streamlit hello를 실행해 보세요
  • 알록달록한 예제 앱이 브라우저에 뜨면 설치 성공 → 이제 streamlit run main.py
  • 끝낼 때는 터미널에서 Ctrl + C (맥도 Ctrl, ⌘ 아님) 또는 터미널 창 닫기
⚠️ 자주 막히는 곳 세 가지
  • “streamlit 명령을 못 찾음”(command not found / 인식되지 않습니다) → 앞에 python3 -m(맥)·python -m(윈도우)를 붙여 python -m streamlit run main.py로 실행
  • 윈도우에서 python이 안 먹힘 → 설치할 때 “Add Python to PATH”를 안 눌렀을 때입니다. 파이썬을 지우고 그 체크를 켠 채 다시 설치
  • “main.py를 못 찾음” → cd로 이동한 폴더에 main.py가 없는 경우. 코드 파일과 데이터 파일이 같은 폴더(다운로드 폴더)에 있는지 확인
① 막히면 AI에게 — 그대로 복사해서 붙여넣기
내 컴퓨터에서만 돌아가는 스트림릿 앱을 실행하려고 해.
내 컴퓨터는 (맥 / 윈도우 중 하나) 이고, 파이썬은 아직 설치 안 했어.
데이터가 인터넷에 올리면 안 되는 민감한 파일이라, 클라우드에 배포하지 않고
내 컴퓨터에서만 열 거야.

파이썬 설치부터 스트림릿 실행까지, 터미널에 그대로 붙여넣을 명령을
순서대로 하나씩 알려줘.
중간에 에러가 나면 그 에러 메시지를 그대로 다시 보낼 테니, 그때 다음에
뭘 하면 되는지 알려줘.
  • 이 강의 내내 한 바이브 코딩과 똑같습니다 — 막히면 에러를 그대로 복사해 AI에게 물어보면, 내 컴퓨터(맥/윈도우)에 딱 맞는 다음 한 줄을 알려줍니다
🔒 민감한 데이터, 이것만은
  • 데이터 파일은 앱 폴더에 그냥 두고 읽기 · 깃허브에 올리거나 인터넷에 업로드하지 않기
  • 이 앱은 Streamlit 클라우드에 배포하지 않기 — 배포하면 데이터를 클라우드로 올려야 합니다. 로컬 실행 자체가 보안입니다
  • API 키가 필요하면 3·4일차처럼 로컬 .streamlit/secrets.toml에만 둡니다 — 이 파일은 내 컴퓨터에만 있고 밖으로 안 나갑니다
  • 실행하면 터미널에 Local URL(내 컴퓨터 전용)과 Network URL(같은 와이파이의 다른 기기용)이 뜹니다 → 민감한 데이터면 Network URL은 공유하지 마세요. localhost:8501만 씁니다
  • 다 보고 나면 터미널에서 Ctrl + C로 종료 → 앱을 끄면 데이터 접근도 함께 끝납니다
