2. 구글시트 연동 앱 — 데이터를 쌓아 보자

  • 만들 것 = 약속 잡기/투표 앱
  • 이름 + 가능시간 제출 → 시트에 한 줄씩 쌓임

2-1. 로봇 직원(서비스 계정) 고용하기

  • 앱이 내 시트에 쓰려면 출입 권한 필요
  • 내 아이디·비번 넣기 = 위험 → 서비스 계정 사용
💡 서비스 계정 = 로봇 직원
  • 내 앱 전용 로봇 직원
  • 자기 이메일(...@....iam.gserviceaccount.com) + 신분증 JSON 키 보유
  • 시트를 로봇 이메일과 공유 → 로봇이 대신 읽고 씀 (내 개인 계정 무관)

2-2. 클릭 순서 (아주 자세히)

  • 진행 위치 = 구글 클라우드 콘솔(console.cloud.google.com)
  1. 프로젝트 만들기
    왼쪽 위 프로젝트 선택 → 새 프로젝트 → 이름(my-sheet-app) → 만들기 → 선택 확인
  2. API 두 개 켜기
    API 및 서비스 → 라이브러리 → Google Sheets API 사용 설정 → Google Drive API도 사용 설정 (둘 다 필수!)
  3. 서비스 계정 만들기
    사용자 인증 정보 → 만들기 → 서비스 계정 → 이름(sheet-bot) → 완료 → 로봇 이메일 생성
  4. JSON 키 다운로드
    서비스 계정 클릭 → 키 → 키 추가 → 새 키 → JSON → 만들기 → .json 다운로드. 로봇 신분증, 공유 금지
  5. 시트를 로봇과 공유
    시트 생성(약속시트) → 공유 → 서비스 계정 이메일 붙여넣기 → 편집자 권한. 빠뜨리면 앱이 시트 못 봄
⚠️ 가장 흔한 실수 두 가지
  • ① Sheets·Drive API 둘 다 안 켬
  • ② 시트를 서비스 계정 이메일과 공유 안 함
  • → 막히면 이 둘부터 확인

2-4. AI에게 앱 주문하기 (①②③)

  • 핵심 두 줄 → 읽기 conn.read() · 저장 conn.update()
① 샘플 프롬프트
스트림릿(streamlit) '약속 잡기' 앱 하나(main.py)를 만들어줘. 조건:

- st-gsheets-connection 라이브러리로 구글시트를 데이터 저장소로 써.
- 화면에 이름 입력창과 '가능한 시간' 선택(여러 개 체크 가능),
  그리고 '제출' 버튼을 둬.
- 제출을 누르면 그 사람의 이름과 가능한 시간을 구글시트에 한 줄 추가해줘.
- 아래에는 지금까지 제출된 전체 명단을 표로 보여줘.
- 시트를 읽을 때는 최신 내용이 바로 보이도록 항상 새로 읽어줘(ttl=0).
- 한국어 라벨, 따뜻한 톤, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.
  • AI가 아래 모습의 코드 제공 (똑같지 않아도 OK)
② AI가 준 코드 (예시)
import streamlit as st
import pandas as pd
from streamlit_gsheets import GSheetsConnection

st.title("🗓️ 우리 언제 모일까?")

# 구글시트 연결 (secrets.toml의 [connections.gsheets] 사용)
conn = st.connection("gsheets", type=GSheetsConnection)

# 지금까지 쌓인 데이터 읽기 — ttl=0 이면 항상 최신본을 새로 읽음
df = conn.read(ttl=0)

# 입력 화면
name = st.text_input("이름")
times = st.multiselect("가능한 시간", ["월", "화", "수", "목", "금"])

if st.button("제출"):
    # 새 한 줄을 만들어 기존 표 아래에 이어 붙이기
    new_row = pd.DataFrame([{"이름": name, "가능시간": ", ".join(times)}])
    updated = pd.concat([df, new_row], ignore_index=True)
    conn.update(data=updated)   # 구글시트에 저장!
    st.success("제출했어요! 아래 명단에 추가됐어요.")

st.subheader("지금까지 제출한 사람들")
st.dataframe(conn.read(ttl=0))
  • 배포 → 제출 → 구글시트 열면 실제로 줄이 쌓임
  • 마음에 안 들면 대화로 개선 ↓
③ 프롬프트 개선 (예시)
좋아! 이어서 고쳐줘.
- 이름을 안 적고 제출하면 '이름을 입력해 주세요' 경고를 띄워줘.
- 명단 맨 아래에 '가장 많이 가능한 요일'을 계산해서 크게 보여줘.
- 표를 요일별 참석 인원 막대그래프로도 보여줘.
✔️ 방금 무슨 일이?
  • 앱이 기억을 갖게 됨 (새로고침·타인 접속에도 데이터 유지)
  • 앱 ↔ 저장소 연결 = 진짜 서비스의 첫걸음

3. 머신러닝 맛보기 — 직접 만드는 작은 AI

  • 지금까지 = 남이 만든 AI 사용
  • 이번 = 내 데이터로 작은 AI 직접 학습

3-1. AI · 머신러닝 · 딥러닝 · LLM (포함 관계)

🧭 AI ⊃ 머신러닝 ⊃ 딥러닝 ⊃ LLM
  • AI = 가장 큰 상자
  • 머신러닝 = 데이터로 스스로 규칙 학습
  • 딥러닝 = 신경망 흉내
  • LLM = 거대 언어모델(챗봇)
  • 오늘 = 기본인 머신러닝 직접 실습
종류정답이 있나요?예시
지도학습있음 (정답표로 배움)연도→기온 예측, 사진→고양이/강아지
비지도학습없음 (스스로 묶음)비슷한 손님끼리 자동 그룹핑
  • 오늘 기온 예측기 = 지도학습(연도→기온 정답표)
  • 군집(KMeans) = 비지도학습 대표 (개념만)

3-2. 머신러닝 5단계

단계하는 일
① 수집데이터 모으기 (서울 연평균기온 표)
② 전처리빠진 값·이상값 다듬기
③ 학습규칙 배우기 (model.fit())
④ 평가점수 매기기 (R²)
⑤ 예측새 입력에 답 (model.predict())

3-3. 2050년 서울 기온 예측기 (①②③)

  • 데이터 = 1일차 기상청 기온(data.kma.go.kr) seoul.csv 재사용 · 빠른 실습은 예제 저장소 주소 그대로

① 샘플 프롬프트
스트림릿(streamlit) '서울 기온 예측기' 앱 하나(main.py)를 만들어줘. 조건:

- 데이터는 이 주소에서 불러와(연도 year, 평균기온 temp 열이 있음):
  https://raw.githubusercontent.com/greatsong/modeudata-with-ai-examples/main/data/seoul.csv
- scikit-learn의 LinearRegression으로 '연도 → 평균기온' 관계를 학습(fit)해줘.
- 화면 위에 R²(결정계수) 값을 카드로 보여주고, 이 값이 뭔지 한 줄로 설명해줘.
- 슬라이더로 1900~2100년 사이 연도를 고르면 예측 기온을 크게 보여줘.
- 실제 데이터 점들과 학습한 직선을 plotly 그래프로 함께 그려줘.
- 훈련 데이터 범위(예: 최근 100년) 밖의 미래 예측에는 '참고용, 조심'
  이라는 안내 문구를 함께 띄워줘.
- 한국어 라벨, 따뜻한 톤, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.
  • 핵심 → model.fit() 딱 한 줄 = ‘학습’ (여기서 AI 탄생)
② AI가 준 코드 (예시)
import streamlit as st
import pandas as pd
from sklearn.linear_model import LinearRegression

st.title("🌡️ 서울 기온 예측기")

# ① 수집 — CSV 불러오기
url = "https://raw.githubusercontent.com/greatsong/modeudata-with-ai-examples/main/data/seoul.csv"
df = pd.read_csv(url)

# ② 전처리 — 입력 X(연도)와 정답 y(기온) 나누기
X = df[["year"]]   # 대괄호 두 개: 표 모양으로
y = df["temp"]

# ③ 학습 — 이 한 줄이 '학습'! 데이터에 가장 잘 맞는 직선을 찾음
model = LinearRegression()
model.fit(X, y)

# ④ 평가 — R²(결정계수): 1에 가까울수록 잘 맞음
r2 = model.score(X, y)
st.metric("R² (얼마나 잘 맞나)", f"{r2:.2f}")

# ⑤ 예측 — 슬라이더로 연도를 넣으면 기온 예측
year = st.slider("예측할 연도", 1900, 2100, 2050)
pred = model.predict([[year]])[0]
st.subheader(f"{year}년 예측 평균기온: {pred:.1f}℃")

if year > int(df["year"].max()):
    st.warning("⚠️ 이 값은 데이터에 없는 미래예요. 참고용으로만, 조심해서 보세요!")
⚠️ 외삽(外揷) — 미래 예측엔 ‘조심’ 딱지
  • 외삽 = 직선을 늘여 데이터 없는 미래(2050) 예측
  • 실제 미래는 직선대로 안 갈 수 있음
  • → 미래 예측 옆엔 항상 ‘조심’ 딱지 · 숫자 맹신 금지
③ 프롬프트 개선 (예시)
좋아! 이어서 고쳐줘.
- 예측 기온 옆에, 지금(가장 최근 해)보다 몇 도 오르는지 함께 보여줘.
- 그래프의 실제 점은 회색, 학습한 직선은 주황색으로.
- 슬라이더를 2100년까지 밀면 경고 문구가 더 크게 뜨도록 해줘.
