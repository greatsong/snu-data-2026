2. Supabase 연동 앱 — 데이터를 쌓아 보자

  • 만들 것 = 약속 잡기/투표 앱 · 이름 + 가능시간 제출 → DB에 한 줄씩 쌓임
  • Supabase = 앱을 위해 만들어진 무료 DB · 카드 등록 없이 · 키 두 줄이면 연결
💡 왜 구글시트 말고 Supabase?
  • 구글시트는 서비스계정 · JSON 키 · 시트 공유 3단 셋업이 완전입문에 최대 난관
  • Supabase는 URL · 키 문자열 2개 복붙이면 끝 — '로봇 계정'·'공유' 개념 없음
  • 구글시트 방식은 이 절 끝에서 강사 데모로 잠깐 구경만

2-1. Supabase 프로젝트·테이블 만들기 (클릭)

  • 진행 위치 = supabase.com · 카드 등록 없음 · 무료
  1. 가입 + 새 프로젝트
    GitHub/이메일로 가입(카드 X) → New project → 이름 + DB 비밀번호 정하고 생성. 준비 1~2분
  2. 테이블 만들기
    왼쪽 Table Editor → New table → 이름 votes → 열 추가 name(text)·times(text) → 저장 (id·시각은 자동)
  3. RLS 잠금 풀기 (연습용)
    새 테이블은 기본 잠김(RLS)이라 그냥 두면 데이터가 안 보여요 → 테이블의 RLS 비활성화(연습용) 또는 '모두 읽기·쓰기' 정책 추가. AI에게 "연습용이니 RLS 꺼는 법 알려줘"라고 물어도 됨
  4. URL·키 복사
    Settings → API → Project URL과 anon public 키 복사. 문자열 2개, 이게 전부!
🔑 키 두 종류만 구분
  • anon(public) 키 = 앱에 넣어도 되는 공개용(RLS가 지킴) → 오늘 이거 사용
  • service_role 키 = 모든 잠금 여는 마스터 → 앱·GitHub 업로드 금지(Secrets에만)

② AI가 준 코드 (예시)
import streamlit as st
import pandas as pd
from supabase import create_client

st.title("🗓️ 우리 언제 모일까?")

# Supabase 연결 — secrets의 URL·키 두 줄이면 끝
sb = create_client(st.secrets["SUPABASE_URL"], st.secrets["SUPABASE_KEY"])

# 입력 화면
name = st.text_input("이름")
times = st.multiselect("가능한 시간", ["월", "화", "수", "목", "금"])

if st.button("제출"):
    # votes 테이블에 새 한 줄 저장!
    sb.table("votes").insert({"name": name, "times": ", ".join(times)}).execute()
    st.success("제출했어요! 아래 명단에 추가됐어요.")

# 지금까지 쌓인 데이터를 읽어 표로
rows = sb.table("votes").select("*").execute().data
st.subheader("지금까지 제출한 사람들")
st.dataframe(pd.DataFrame(rows))
  • 배포 → 제출 → Supabase의 Table Editor 열면 실제로 줄이 쌓임
  • 마음에 안 들면 대화로 개선 ↓

② AI가 준 코드 (예시)
import streamlit as st
import pandas as pd
from sklearn.linear_model import LinearRegression

st.title("🌡️ 서울 기온 예측기")

# ① 수집 — CSV 불러오기
url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/seoul.csv"
df = pd.read_csv(url)

# ② 전처리 — 입력 X(연도)와 정답 y(기온) 나누기
X = df[["year"]]   # 대괄호 두 개: 표 모양으로
y = df["temp"]

# ③ 학습 — 이 한 줄이 '학습'! 데이터에 가장 잘 맞는 직선을 찾음
model = LinearRegression()
model.fit(X, y)

# ④ 평가 — R²(결정계수): 1에 가까울수록 잘 맞음
r2 = model.score(X, y)
st.metric("R² (얼마나 잘 맞나)", f"{r2:.2f}")

# ⑤ 예측 — 슬라이더로 연도를 넣으면 기온 예측
year = st.slider("예측할 연도", 1900, 2100, 2050)
pred = model.predict([[year]])[0]
st.subheader(f"{year}년 예측 평균기온: {pred:.1f}℃")

if year > int(df["year"].max()):
    st.warning("⚠️ 이 값은 데이터에 없는 미래예요. 참고용으로만, 조심해서 보세요!")
⚠️ 외삽(外揷) — 미래 예측엔 ‘조심’ 딱지
  • 외삽 = 직선을 늘여 데이터 없는 미래(2050) 예측
  • 실제 미래는 직선대로 안 갈 수 있음
  • → 미래 예측 옆엔 항상 ‘조심’ 딱지 · 숫자 맹신 금지
③ 프롬프트 개선 (예시)
좋아! 이어서 고쳐줘.

- 그래프의 실제 점은 회색, 학습한 직선은 주황색으로.
- 슬라이더를 2100년까지 밀면 경고 문구가 더 크게 뜨도록 해줘.
