2. Supabase 연동 앱 — 데이터를 쌓아 보자

  • 만들 것 = 약속 잡기/투표 앱 · 이름 + 가능시간 제출 → DB에 한 줄씩 쌓임
  • Supabase = 앱을 위해 만들어진 무료 DB · 카드 등록 없이 · 키 두 줄이면 연결
💡 왜 구글시트 말고 Supabase?
  • 구글시트는 서비스계정 · JSON 키 · 시트 공유 3단 셋업이 완전입문에 최대 난관
  • Supabase는 URL · 키 문자열 2개 복붙이면 끝 — '로봇 계정'·'공유' 개념 없음
  • 구글시트 방식은 이 절 끝에서 강사 데모로 잠깐 구경만

2-1. Supabase 프로젝트·테이블 만들기 (클릭)

  • 진행 위치 = supabase.com · 카드 등록 없음 · 무료
  1. 가입 + 새 프로젝트
    GitHub/이메일로 가입(카드 X) → New project → 이름 + DB 비밀번호 정하고 생성. 준비 1~2분
  2. 테이블 만들기
    왼쪽 Table Editor → New table → 이름 votes → 열 추가 name(text)·times(text) → 저장 (id·시각은 자동)
  3. RLS 잠금 풀기 (연습용)
    새 테이블은 기본 잠김(RLS)이라 그냥 두면 데이터가 안 보여요 → 테이블의 RLS 비활성화(연습용) 또는 '모두 읽기·쓰기' 정책 추가. AI에게 "연습용이니 RLS 꺼는 법 알려줘"라고 물어도 됨
  4. URL·키 복사
    Settings → API Keys에서 publishable 키(예전 이름: anon public)를, Settings → Data API에서 Project URL을 복사. 문자열 2개, 이게 전부! (화면 구성은 조금씩 바뀔 수 있어요 — 못 찾으면 AI에게 물어보기)
🔑 키 두 종류만 구분
  • publishable 키(예전 anon) = 앱에 넣어도 되는 공개용(RLS가 지킴) → 오늘 이거 사용 · 단, 오늘 실습 테이블은 연습용으로 RLS를 꺼 뒀으니 이 보호가 없는 상태 — 실제 서비스에선 RLS를 켠 채 정책으로 허용 범위만 열기
  • secret 키(예전 service_role) = 모든 잠금 여는 마스터 → 앱·GitHub 업로드 금지(Secrets에만)

② AI가 준 코드 (예시)
import streamlit as st
import pandas as pd
from supabase import create_client

st.title("🗓️ 우리 언제 모일까?")

# Supabase 연결 — secrets의 URL·키 두 줄이면 끝
sb = create_client(st.secrets["SUPABASE_URL"], st.secrets["SUPABASE_KEY"])

# 입력 화면
name = st.text_input("이름")
times = st.multiselect("가능한 시간", ["월", "화", "수", "목", "금"])

if st.button("제출"):
    # votes 테이블에 새 한 줄 저장!
    sb.table("votes").insert({"name": name, "times": ", ".join(times)}).execute()
    st.success("제출했어요! 아래 명단에 추가됐어요.")

# 지금까지 쌓인 데이터를 읽어 표로
rows = sb.table("votes").select("*").execute().data
st.subheader("지금까지 제출한 사람들")
st.dataframe(pd.DataFrame(rows))
  • 배포 → 제출 → Supabase의 Table Editor 열면 실제로 줄이 쌓임
  • 마음에 안 들면 대화로 개선 ↓

import streamlit as st
import pandas as pd
from sklearn.linear_model import LinearRegression

st.title("🌡️ 서울 기온 예측기")

# ① 수집 — CSV 불러오기 (1907년~최근, 일별 기온 · 매일 자동 갱신)
url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/seoul.csv"
df = pd.read_csv(url)

# ② 전처리 — 연도별 평균으로 묶기 · 진행 중인 올해와
#            관측일이 부족한 해(전쟁 시기 등)는 빼기
df["연도"] = pd.to_datetime(df["날짜"]).dt.year
yearly = df.groupby("연도")["평균기온"].agg(["mean", "count"]).reset_index()
yearly = yearly[(yearly["연도"] < df["연도"].max()) & (yearly["count"] >= 300)]
yearly = yearly.rename(columns={"mean": "평균기온"})

X = yearly[["연도"]]   # 대괄호 두 개: 표 모양으로
y = yearly["평균기온"]

# ③ 학습 — 이 한 줄이 '학습'! 데이터에 가장 잘 맞는 직선을 찾음
model = LinearRegression()
model.fit(X, y)

# ④ 평가 — R²(결정계수): 1에 가까울수록 잘 맞음
r2 = model.score(X, y)
st.metric("R² (얼마나 잘 맞나)", f"{r2:.2f}")

# (지면상 생략 — 실제 코드엔 plotly로 그린 '실제 점 + 학습 직선' 그래프가 함께 옵니다)

# ⑤ 예측 — 슬라이더로 연도를 넣으면 기온 예측
year = st.slider("예측할 연도", 1900, 2100, 2050)
pred = model.predict([[year]])[0]
st.subheader(f"{year}년 예측 평균기온: {pred:.1f}℃")

if year > int(yearly["연도"].max()):
    st.warning("⚠️ 이 값은 데이터에 없는 미래예요. 참고용으로만, 조심해서 보세요!")
⚠️ 외삽(外揷) — 미래 예측엔 ‘조심’ 딱지
  • 외삽 = 직선을 늘여 데이터 없는 미래(2050) 예측
  • 실제 미래는 직선대로 안 갈 수 있음
  • → 미래 예측 옆엔 항상 ‘조심’ 딱지 · 숫자 맹신 금지

  • 데이터 모아 스스로 규칙 학습
  • 규칙으로 새 예측 산출
  • 규모만 작을 뿐, 원리는 거대 AI와 동일 = 머신러닝 직접 실습

3-4. 한 걸음 더 (추가 학습) — 영화 흥행 예측기 (①②③)

🎬 왜 영화도 해 보나
  • 기온 예측기는 입력이 연도 하나 · 영화는 여러 조건(스크린수·상영횟수·첫날 관객)으로 예측 — 입력이 여럿인 머신러닝을 경험
  • 흥행 데이터는 천만 영화 한 편이 판을 다 흔드는 성질 · 그대로 학습하면 큰 영화에만 끌려감
  • 여유가 있으면 도전 · 시간이 빠듯하면 오늘 밤 숙제로
  • 데이터 = 예제 저장소 주소 그대로(kobis.csv · 최근 개봉한 영화들 · 원자료는 영화진흥위원회 kobis.or.kr) — 다운로드 없이 주소로 바로 불러오기
  • 열 = 영화명·개봉일·스크린수·상영횟수·순위·관객수(첫날)·최종관객 · 목표 = 초반 지표로 최종관객 예측
① 샘플 프롬프트
스트림릿(streamlit) '영화 흥행 예측기' 앱 하나(main.py)를 만들어줘. 조건:

- 데이터는 이 주소에서 불러와(최근 개봉한 영화들이야.
  열은 영화명, 개봉일, 스크린수, 상영횟수, 순위, 관객수(개봉 첫날), 최종관객):
  https://raw.githubusercontent.com/greatsong/modudata/main/data/kobis.csv
- 스크린수, 상영횟수, 첫날 관객수(관객수 열)로 최종관객을 예측하는
  선형회귀 모델을 scikit-learn으로 만들어줘.
- 흥행 숫자는 편차가 아주 커서 몇몇 대박 영화가 결과를 좌우해.
  그러니 입력과 정답을 로그로 바꿔서 학습하고, 예측값은 다시 원래 크기로 되돌려 보여줘.
- 화면 위에 R²(결정계수) 값을 카드로 보여주고, 이 값이 뭔지 한 줄로 설명해줘.
- 스크린수·상영횟수·첫날 관객수를 슬라이더나 입력창으로 넣으면
  예측 최종관객을 크게 보여줘.
- 필요한 라이브러리 목록(requirements.txt)도 같이 줘.
  버전 숫자 없이 이름만, 스트림릿·판다스·넘파이는 기본 설치라 빼고.
- 한국어 라벨, 따뜻한 톤, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.
  • 핵심 → 여기서도 model.fit() 한 줄 · 다만 입력이 여러 개이고 로그로 바꿔 학습
② AI가 준 코드 (예시)
import streamlit as st
import pandas as pd
import numpy as np
from sklearn.linear_model import LinearRegression

st.title("🎬 영화 흥행 예측기")

# ① 수집 — CSV 불러오기 (최근 개봉 영화)
url = "https://raw.githubusercontent.com/greatsong/modudata/main/data/kobis.csv"
df = pd.read_csv(url)

# ② 전처리 — 입력 X, 정답 y · 흥행은 편차가 커서 로그로 다룸
X = df[["스크린수", "상영횟수", "관객수"]]   # 관객수 = 개봉 첫날
y = df["최종관객"]
Xlog, ylog = np.log1p(X), np.log1p(y)

# ③ 학습 — 이 한 줄이 '학습'! 입력이 여러 개인 것만 빼면 똑같음
model = LinearRegression()
model.fit(Xlog, ylog)

# ④ 평가 — R²(결정계수): 1에 가까울수록 잘 맞음
r2 = model.score(Xlog, ylog)
st.metric("R² (얼마나 잘 맞나)", f"{r2:.2f}")

# ⑤ 예측 — 세 값을 넣으면 최종관객 예측 (로그를 다시 원래 크기로)
screens = st.number_input("스크린수", 1, 3000, 1200)
shows   = st.number_input("상영횟수", 1, 20000, 4500)
day1    = st.number_input("첫날 관객수", 1, 2000000, 110000)
pred = np.expm1(model.predict(np.log1p([[screens, shows, day1]])))[0]
st.subheader(f"예측 최종관객: 약 {int(pred):,}명")
⚠️ 왜 로그로 바꿨을까 — 이상치와 척도
  • 천만 영화 한 편이 있으면 회귀 직선이 그쪽으로 확 쏠림 · 작은 영화 예측이 엉망
  • 로그로 바꾸면 큰 영화와 작은 영화가 비슷한 무게로 · 실측: 로그 없이 R²≈0.32 → 로그 쓰면 0.76~0.83
  • 같은 데이터라도 어떤 척도로 보느냐가 결과를 바꾼다 — 데이터를 다루는 눈
③ 프롬프트 개선 (예시)
좋아! 이어서 고쳐줘.
- 예측 최종관객이 '천만 돌파'면 축하 문구를, 백만 미만이면 다른 문구를 띄워줘.
- 실제 최종관객과 예측값을 비교하는 산점도를 그려줘(대각선이 완벽한 예측).
- 스크린수 하나만 썼을 때와 세 개 다 썼을 때 R²가 어떻게 다른지 보여줘.
