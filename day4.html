오늘의 약속 — 데이터를 어딘가에 쌓고, 그 데이터로 미래를 살짝 예측해 봅니다. 여전히 코드는 AI가 짜고, 우리는 주문하고 확인해요.

0. 어제까지 온 길 (짧게 복습)

3일차까지 우리는 AI를 앱 안에 넣는 법을 배웠어요. 딱 네 단어만 기억하면 됩니다.

단어한 줄 뜻
API다른 회사의 AI·데이터를 불러다 쓰는 창구
API 키그 창구를 쓰는 내 출입증 (남에게 보이면 안 됨)
secrets키를 코드 밖 비밀 금고에 넣어두는 곳
채팅앱사용자 질문 → AI 응답을 주고받는 앱

그런데 어제 만든 채팅앱에는 한 가지 아쉬움이 있었어요. 대화가 저장되지 않는다는 것. 오늘은 그 문제부터 풀어 봅니다.

1. 왜 ‘저장’이 필요할까?

스트림릿 앱은 화면에 값을 잠깐 들고 있어요. 이 임시 기억을 session_state라고 부릅니다. 그런데 이 기억에는 결정적인 약점이 있어요.

⚠️ 새로고침하면 사라집니다 (휘발성)

session_state에 담은 값은 브라우저를 새로고침하거나 닫으면 통째로 날아가요. 마치 화이트보드에 적은 메모 같은 거예요. 잠깐은 보이지만, 지우면 끝. 다른 사람이 접속하면 그 사람에겐 아예 안 보이고요.

그래서 여러 사람이 남긴 데이터를 계속 쌓으려면 앱 바깥에 있는 진짜 저장소가 필요해요. 저장소에도 종류가 있는데, 지도로 보면 이렇습니다.

저장소난이도느낌
구글시트쉬움 🐣우리가 아는 그 스프레드시트. 눈에 보이고 손으로도 고침 → 오늘 이걸로!
진짜 데이터베이스(DB)어려움대용량·빠른 검색용 전문 저장소 → 5일차에 맛보기

오늘은 가장 친근한 구글시트를 데이터 저장소로 씁니다. 우리가 매일 보던 표가 앱의 ‘창고’가 되는 거예요.

2. 구글시트 연동 앱 — 데이터를 쌓아 보자

만들 앱은 약속 잡기/투표 앱이에요. 사용자가 이름 + 가능한 시간을 제출하면, 그 내용이 구글시트에 한 줄씩 쌓입니다. 다 같이 채운 시트를 보면 언제 모이면 좋을지 한눈에 보이죠.

오늘 세팅 중 이 부분이 가장 무겁습니다. 하지만 클릭 순서만 그대로 따라오면 돼요. 천천히 같이 가요. 🐢

2-1. 로봇 직원(서비스 계정) 고용하기

앱이 내 구글시트에 글을 쓰려면 ‘출입 권한’이 필요해요. 그런데 내 구글 아이디·비밀번호를 앱에 넣는 건 위험하죠. 그래서 구글은 서비스 계정(service account)이라는 걸 줍니다.

💡 서비스 계정 = 로봇 직원

서비스 계정은 내 앱만을 위한 로봇 직원이에요. 이 로봇에게는 ...@....iam.gserviceaccount.com 같은 자기 이메일 주소와, 신분증 역할을 하는 JSON 키 파일이 있어요. 내가 구글시트를 이 로봇의 이메일과 ‘공유’해 주면, 로봇이 시트에 대신 읽고 씁니다. 내 개인 계정은 건드리지 않아요.

2-2. 클릭 순서 (아주 자세히)

구글 클라우드 콘솔(console.cloud.google.com)에서 진행합니다. 순서대로만 누르세요.

  1. 프로젝트 만들기
    콘솔 왼쪽 위 프로젝트 선택 → 새 프로젝트 → 이름(예: my-sheet-app) → 만들기. 만든 프로젝트가 선택돼 있는지 확인.
  2. API 두 개 켜기
    왼쪽 메뉴 API 및 서비스 → 라이브러리. 검색창에 Google Sheets API → 열어서 사용 설정. 다시 라이브러리로 가서 Google Drive API도 똑같이 사용 설정. (두 개 다 켜야 해요!)
  3. 서비스 계정 만들기
    API 및 서비스 → 사용자 인증 정보 → 위쪽 사용자 인증 정보 만들기 → 서비스 계정. 이름(예: sheet-bot)을 적고 완료. 목록에 로봇 이메일이 하나 생겨요.
  4. JSON 키 다운로드
    방금 만든 서비스 계정을 클릭 → 키 탭 → 키 추가 → 새 키 만들기 → JSON → 만들기. 작은 .json 파일이 컴퓨터에 내려받아집니다. 이게 로봇의 신분증이에요. 아무에게도 주지 마세요.
  5. 시트를 로봇과 공유
    데이터를 쌓을 구글시트를 하나 만들고(예: 약속시트), 오른쪽 위 공유 → 서비스 계정 이메일(...@....iam.gserviceaccount.com)을 붙여넣고 편집자 권한으로 공유. 이 단계를 빠뜨리면 앱이 시트를 못 봐요.

⚠️ 가장 흔한 실수 두 가지

① Sheets API·Drive API를 켜지 않음 — 둘 다 켜야 합니다. ② 시트를 서비스 계정 이메일과 공유하지 않음 — 로봇이 문 앞에서 막혀요. 막히면 이 두 가지부터 확인!

2-3. 비밀 금고에 키 넣기 + 라이브러리

스트림릿에서 구글시트를 쓰려면 st-gsheets-connection이라는 라이브러리가 필요해요. 어제처럼 requirements.txt에 적어 둡니다.

requirements.txt
streamlit
st-gsheets-connection
pandas

이제 3일차에 배운 비밀 금고(.streamlit/secrets.toml)에 다운로드한 JSON 키의 내용을 옮겨 담아요. 아래 모양이면 됩니다(값은 여러분 JSON 파일에서 그대로 복사).

.streamlit/secrets.toml (모양)
[connections.gsheets]
spreadsheet = "여기에_구글시트_주소_붙여넣기"
type = "service_account"
project_id = "내-프로젝트-id"
private_key_id = "…"
private_key = "-----BEGIN PRIVATE KEY-----\n…\n-----END PRIVATE KEY-----\n"
client_email = "sheet-bot@…iam.gserviceaccount.com"
client_id = "…"
auth_uri = "https://accounts.google.com/o/oauth2/auth"
token_uri = "https://oauth2.googleapis.com/token"
⚠️ 여기서 잘 막혀요

private_key는 따옴표로 감싸고, 줄바꿈은 \n 그대로 두세요. 각 줄 들여쓰기·따옴표가 하나만 어긋나도 연결이 안 돼요. 클라우드에 배포할 때는 이 내용을 스트림릿 사이트의 Settings → Secrets에 똑같이 붙여넣습니다.

2-4. AI에게 앱 주문하기 (①②③)

이제 재료가 다 준비됐어요. AI에게 앱을 주문합니다. 읽고 쓰는 핵심은 딱 두 줄이에요 — 읽기는 conn.read(), 저장은 conn.update().

① 샘플 프롬프트
스트림릿(streamlit) '약속 잡기' 앱 하나(main.py)를 만들어줘. 조건:

- st-gsheets-connection 라이브러리로 구글시트를 데이터 저장소로 써.
- 화면에 이름 입력창과 '가능한 시간' 선택(여러 개 체크 가능),
  그리고 '제출' 버튼을 둬.
- 제출을 누르면 그 사람의 이름과 가능한 시간을 구글시트에 한 줄 추가해줘.
- 아래에는 지금까지 제출된 전체 명단을 표로 보여줘.
- 시트를 읽을 때는 최신 내용이 바로 보이도록 항상 새로 읽어줘(ttl=0).
- 한국어 라벨, 따뜻한 톤, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.

잠깐 기다리면 AI가 아래 모습의 코드를 줍니다. 여러분 것과 똑같지 않아도 괜찮아요.

② AI가 준 코드 (예시)
import streamlit as st
import pandas as pd
from streamlit_gsheets import GSheetsConnection

st.title("🗓️ 우리 언제 모일까?")

# 구글시트 연결 (secrets.toml의 [connections.gsheets] 사용)
conn = st.connection("gsheets", type=GSheetsConnection)

# 지금까지 쌓인 데이터 읽기 — ttl=0 이면 항상 최신본을 새로 읽음
df = conn.read(ttl=0)

# 입력 화면
name = st.text_input("이름")
times = st.multiselect("가능한 시간", ["월", "화", "수", "목", "금"])

if st.button("제출"):
    # 새 한 줄을 만들어 기존 표 아래에 이어 붙이기
    new_row = pd.DataFrame([{"이름": name, "가능시간": ", ".join(times)}])
    updated = pd.concat([df, new_row], ignore_index=True)
    conn.update(data=updated)   # 구글시트에 저장!
    st.success("제출했어요! 아래 명단에 추가됐어요.")

st.subheader("지금까지 제출한 사람들")
st.dataframe(conn.read(ttl=0))

배포해서 켠 뒤, 이름을 넣고 제출해 보세요. 구글시트를 열어 보면 방금 넣은 줄이 실제로 쌓여 있어요! 마음에 안 들면 대화로 고칩니다.

③ 프롬프트 개선 (예시)
좋아! 이어서 고쳐줘.
- 이름을 안 적고 제출하면 '이름을 입력해 주세요' 경고를 띄워줘.
- 명단 맨 아래에 '가장 많이 가능한 요일'을 계산해서 크게 보여줘.
- 표를 요일별 참석 인원 막대그래프로도 보여줘.
✔️ 방금 무슨 일이 일어났나요?

이제 여러분의 앱은 기억을 갖게 됐어요. 새로고침해도, 다른 사람이 접속해도 데이터가 남아 있죠. 앱과 저장소를 연결한 것 — 이게 진짜 서비스의 첫걸음이에요.

3. 머신러닝 맛보기 — 직접 만드는 작은 AI

지금까지 우리는 남이 만든 AI를 불러다 썼어요. 이번엔 내가 데이터로 직접 아주 작은 AI를 하나 학습시켜 봅니다. 그전에 지도부터 그려요.

3-1. AI · 머신러닝 · 딥러닝 · LLM (포함 관계)

이 단어들은 큰 상자 안에 작은 상자가 든 관계예요.

🧭 AI ⊃ 머신러닝 ⊃ 딥러닝 ⊃ LLM

AI(인공지능)가 가장 큰 상자. 그 안에 머신러닝(데이터로 스스로 규칙을 배우는 방식)이 있고, 그 안에 딥러닝(뇌 신경망을 흉내 낸 방식)이 있고, 그 안에 우리가 매일 쓰는 LLM(챗봇 같은 거대 언어모델)이 있어요. 오늘은 가장 기본인 머신러닝을 직접 해 봅니다.

머신러닝은 크게 두 갈래예요.

종류정답이 있나요?예시
지도학습있음 (정답표로 배움)연도→기온 예측, 사진→고양이/강아지 구분
비지도학습없음 (스스로 묶음)비슷한 손님끼리 자동 그룹핑

오늘 만들 기온 예측기는 ‘연도→기온’이라는 정답표로 배우는 지도학습이에요. (정답 없이 비슷한 것끼리 묶는 군집(KMeans)은 비지도학습의 대표 예 — 개념만 알아두세요.)

3-2. 머신러닝 5단계

어떤 머신러닝이든 흐름은 똑같아요. 요리 순서처럼요.

단계하는 일
① 수집데이터를 모은다 (서울 연평균기온 표)
② 전처리빠진 값·이상한 값을 다듬는다
③ 학습데이터에서 규칙을 배운다 (model.fit())
④ 평가얼마나 잘 맞추는지 점수를 매긴다 (R²)
⑤ 예측새 입력에 답을 내놓는다 (model.predict())

3-3. 2050년 서울 기온 예측기 (①②③)

서울의 연도별 평균기온 데이터로 직선 하나를 학습시켜, 슬라이더로 연도를 넣으면 예측 기온이 나오는 앱을 만듭니다. 사용할 도구는 scikit-learn의 LinearRegression(선형회귀 = 데이터에 가장 잘 맞는 직선 긋기)이에요.

requirements.txt
streamlit
pandas
scikit-learn
plotly
① 샘플 프롬프트
스트림릿(streamlit) '서울 기온 예측기' 앱 하나(main.py)를 만들어줘. 조건:

- 데이터는 이 주소에서 불러와(연도 year, 평균기온 temp 열이 있음):
  https://raw.githubusercontent.com/greatsong/modeudata-with-ai-examples/main/data/seoul.csv
- scikit-learn의 LinearRegression으로 '연도 → 평균기온' 관계를 학습(fit)해줘.
- 화면 위에 R²(결정계수) 값을 카드로 보여주고, 이 값이 뭔지 한 줄로 설명해줘.
- 슬라이더로 1900~2100년 사이 연도를 고르면 예측 기온을 크게 보여줘.
- 실제 데이터 점들과 학습한 직선을 plotly 그래프로 함께 그려줘.
- 훈련 데이터 범위(예: 최근 100년) 밖의 미래 예측에는 '참고용, 조심'
  이라는 안내 문구를 함께 띄워줘.
- 한국어 라벨, 따뜻한 톤, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.

핵심은 아래 코드에서 model.fit() 딱 한 줄이 ‘학습’이라는 점이에요. 이 한 줄에서 AI가 만들어집니다.

② AI가 준 코드 (예시)
import streamlit as st
import pandas as pd
from sklearn.linear_model import LinearRegression

st.title("🌡️ 서울 기온 예측기")

# ① 수집 — CSV 불러오기
url = "https://raw.githubusercontent.com/greatsong/modeudata-with-ai-examples/main/data/seoul.csv"
df = pd.read_csv(url)

# ② 전처리 — 입력 X(연도)와 정답 y(기온) 나누기
X = df[["year"]]   # 대괄호 두 개: 표 모양으로
y = df["temp"]

# ③ 학습 — 이 한 줄이 '학습'! 데이터에 가장 잘 맞는 직선을 찾음
model = LinearRegression()
model.fit(X, y)

# ④ 평가 — R²(결정계수): 1에 가까울수록 잘 맞음
r2 = model.score(X, y)
st.metric("R² (얼마나 잘 맞나)", f"{r2:.2f}")

# ⑤ 예측 — 슬라이더로 연도를 넣으면 기온 예측
year = st.slider("예측할 연도", 1900, 2100, 2050)
pred = model.predict([[year]])[0]
st.subheader(f"{year}년 예측 평균기온: {pred:.1f}℃")

if year > int(df["year"].max()):
    st.warning("⚠️ 이 값은 데이터에 없는 미래예요. 참고용으로만, 조심해서 보세요!")
⚠️ 외삽(外揷) — 미래 예측엔 ‘조심’ 딱지

직선을 그대로 늘여 데이터에 없는 미래(2050년)를 맞히는 걸 외삽이라고 해요. 과거 흐름을 참고할 수는 있지만, 실제 미래는 직선대로 가지 않을 수 있어요. 그래서 미래 예측 옆에는 항상 ‘조심’ 딱지를 붙입니다. 숫자를 맹신하지 않기!

결과를 본 뒤, 대화로 더 키워 봐요.

③ 프롬프트 개선 (예시)
좋아! 이어서 고쳐줘.
- 예측 기온 옆에, 지금(가장 최근 해)보다 몇 도 오르는지 함께 보여줘.
- 그래프의 실제 점은 회색, 학습한 직선은 주황색으로.
- 슬라이더를 2100년까지 밀면 경고 문구가 더 크게 뜨도록 해줘.
💡 방금 여러분이 한 일

데이터를 모아 스스로 규칙을 배우게(학습) 하고, 그 규칙으로 새 예측을 뽑았어요. 규모는 작지만, 원리는 세상의 거대한 AI와 똑같아요. 여러분은 방금 머신러닝을 직접 해 본 거예요.
