쌓고 예측하기 — 강사 진행안

수강생용 4일차 페이지 ↗ 를 화면에 띄운 채 진행합니다. 여기에는 시간표 · 멘트 · 예상 질문 · 예상 오류 · 이론 심화를 덧댔습니다.

🎙️ 오늘의 목표 선언 (개강 멘트)

“오늘 우리 앱은 기억을 갖게 됩니다 — 데이터를 구글시트에 쌓아요. 그리고 오후엔 직접 만드는 작은 AI로 2050년 서울 기온을 예측해 봅니다. 코드는 여전히 AI가 짜요.”

📖 오늘의 큰 그림 — 저장 → 학습

오전은 ‘휘발성 극복’(session_state → 외부 저장소=구글시트), 오후는 ‘남의 AI 쓰기’에서 ‘내 AI 만들기’로의 전환(LLM 호출 → scikit-learn 학습). 두 축이 오늘의 뼈대입니다. ①②③ 세 박자는 오늘도 유지 — 구글시트 앱과 예측기 각각 ②(샘플 코드)까지 짚어줍니다.

블록별 진행 노트

복습 (0~20′)

🎙️ 멘트 — 어제의 아쉬움에서 출발

“어제 만든 채팅앱, 멋졌죠? 근데 새로고침하면 대화가 싹 사라져요. 오늘은 그걸 고칩니다.” — 실제로 어제 앱을 새로고침해 데이터가 날아가는 걸 눈앞에서 보여주면 동기부여가 확실합니다.

1. 왜 저장이 필요한가 (20~40′)

📖 이론 심화 — 휘발성과 저장소 선택

session_state는 세션(브라우저 탭) 단위의 임시 메모리. 새로고침·재접속·타 사용자에겐 공유되지 않음. 영속 저장이 필요하면 앱 외부 저장소로. 오늘 구글시트를 고르는 이유: 눈에 보이고, 손으로도 고칠 수 있고, 설정이 DB보다 가볍다. 진짜 DB(관계형/문서형)는 5일차 맛보기.

🙋 예상 질문

“왜 시트를 데이터베이스로 써요?” → “쉽고, 눈에 바로 보이니까요. 앱이 쓴 걸 시트에서 직접 확인할 수 있어 초보자에게 최고의 교보재예요. 대용량·빠른 검색이 필요한 진짜 DB는 내일 맛봅니다.”

2. 구글시트 연동 앱 (40~110′) — 오늘의 최대 난관

🎙️ 멘트 — JSON 키 세팅, 같이 천천히

“지금부터 오늘 세팅 부담 1위 구간이에요. 어렵지 않아요, 클릭 순서만 저를 따라오면 됩니다. 화면을 크게 띄울 테니 한 클릭씩 같이 갈게요.” — 강사 화면으로 콘솔 화면을 실시간 미러링하며 진행.

📖 세팅 부담을 낮추는 준비물 (강사 필수)

JSON 키 발급이 교재 전체에서 세팅 부담 1위. 반드시 미리 준비: ① 완성된 시트 템플릿(열 이름까지)을 공유해 복제하게, ② 화면 따라하기(콘솔 클릭을 강사 화면으로 단계별 시연), ③ 실패자용 예비 — 이미 서비스 계정과 연결해 둔 ‘미리 연결된 시트’ 앱을 하나 준비해, 세팅에 막힌 수강생은 그걸로 나머지 실습을 이어가게 합니다. 세팅 실패가 ML 블록 참여를 막지 않도록 방어하는 게 핵심.

🧯 예상 오류 (빈출)

권한 오류(로봇이 시트 못 봄) → 시트를 서비스 계정 이메일과 공유했는지(편집자). 1순위 원인. · API 사용설정 누락 → Google Sheets API·Google Drive API 둘 다 켰는지(Drive 빠뜨리는 경우 많음). · secrets.toml 오류 → private_key의 따옴표·\n·들여쓰기가 어긋남. TOML은 들여쓰기·따옴표에 민감. · 갱신이 안 됨 → conn.read(ttl=0)를 안 줘서 캐시된 옛 데이터가 보임. 제출 후 표가 안 바뀌면 여기부터.

🙋 예상 질문

“JSON 키를 잃어버리면요?” → 콘솔에서 새 키를 다시 발급하면 됨(옛 키는 삭제 권장). · “서비스 계정이 뭐예요?” → “사람 계정이 아니라 앱 전용 로봇 직원이에요. 로봇 이메일로 시트를 공유해 주면 로봇이 대신 읽고 씁니다.”

3. 머신러닝 맛보기 (120~180′)

🎙️ 멘트 — “직접 만드는 작은 AI”

“지금까진 남이 만든 AI를 불러다 썼죠. 이번엔 여러분이 데이터로 직접 아주 작은 AI를 학습시켜요.” 목표는 완벽한 예측이 아니라 ‘학습이란 이런 것’이라는 체험. model.fit() 한 줄에서 “지금 AI가 태어났어요!”로 감탄 포인트를 만듭니다.

📖 이론 심화 — 회귀와 R²

선형회귀는 데이터에 가장 잘 맞는 직선(오차 제곱합 최소)을 찾는 것. R²(결정계수)는 0~1, 모델이 데이터의 변동을 얼마나 설명하는지의 비율 — 1에 가까울수록 설명력이 큼. 단, R²가 높다고 미래 예측이 정확하단 뜻은 아님(훈련 범위 안에서의 적합도일 뿐).

📖 이론 심화 — 과적합 · 외삽 · 상관≠인과

과적합(overfitting): 고차 다항식으로 훈련 점을 과하게 맞추면 훈련 데이터엔 완벽하지만 새 데이터엔 엉망 → 그래서 오늘은 직선(1차)으로 단순하게. · 외삽(extrapolation): 훈련 범위 밖 예측은 위험 — 2050년은 데이터에 없으므로 직선을 늘인 ‘가정’일 뿐. 수강생에게 반드시 ‘조심 딱지’로 각인. · 상관≠인과: 연도와 기온이 함께 오른다고 “연도가 기온을 올린다”는 아님(뒤에 도시화·온실가스 등 실제 원인). 예측기는 패턴 요약이지 원인 규명이 아님을 강조.

🧯 예상 오류

ModuleNotFoundError → requirements.txt에 scikit-learn(설치명, import는 sklearn) 확인. · X 모양 오류 → df[["year"]] 대괄호 두 개(2차원)여야 함, df["year"](1차원)는 에러. AI가 준 코드면 대개 정상. · CSV 열 이름 불일치 → 예제는 year, temp. 다른 열 이름이면 프롬프트에 명시. · predict 경고 → model.predict([[year]]) 형태 유지.

🙋 예상 질문

“예측을 믿어도 돼요?” → “훈련 범위 안은 참고할 만해요. 하지만 2050년 같은 미래는 외삽이라 조심해야 해요. 직선을 늘인 가정일 뿐, 실제 미래는 다르게 갈 수 있어요.” · “군집(KMeans)은 뭐예요?” → “정답 없이 비슷한 것끼리 자동으로 묶는 비지도학습이에요. 오늘은 개념만, 필요하면 5일차 프로젝트에서.”

🌱 씨앗 + 마무리 (180′~)

🎙️ 마무리 멘트 + 씨앗

“오늘 밤 내 프로젝트 주제 한 줄만 정해 오세요. 지난 4일 조각이 전부 재료예요. 숙제는 선택이지만, 적어 오면 내일 오전이 편해요. 내일은 발표 + 더 멋진 프로젝트로 마무리합니다.”
