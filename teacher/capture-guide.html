스크린샷 촬영 가이드

학생용 교재의 설정 단계에는 스크린샷 자리(빗금 박스)가 들어가 있습니다. 아래 목록대로 캡처해 교체하면 완전 입문자용 교재가 완성됩니다. 실제 화면은 강사 계정으로 로그인해야 나오므로 강사가 직접 촬영합니다.

🖼️ 교체 방법 (3단계)

① 화면을 캡처해 web/assets/shots/ 폴더에 shot 코드 이름으로 저장(예: day1-01-new-repo.png).
② 해당 페이지에서 <div class="shot">…</div> 블록을 찾아,
③ 아래 한 줄로 바꿉니다(경로는 강사용이면 ../assets/…):

교체 코드 (예: day1-01)
<img class="shot-img" src="assets/shots/day1-01-new-repo.png"
     alt="새 저장소 만들기 화면">
💡 촬영 팁

브라우저 폭을 적당히 좁혀(≈1000px) 글씨가 크게. 개인정보(이메일·실명·키 값)는 가리거나 더미로. 클릭할 버튼에 빨간 사각형/화살표를 넣으면 초보자가 훨씬 쉬워요.

📖 왜 스크린샷을 강사가 직접?

GitHub·Google Cloud·Streamlit 화면은 로그인한 실제 계정에서만 나오고, UI가 수시로 바뀝니다. 촬영 시점의 최신 화면을 강사가 찍는 것이 가장 정확해요. 2·5일차는 대부분 1일차의 GitHub 기본기를 재사용해 별도 촬영이 적습니다.
