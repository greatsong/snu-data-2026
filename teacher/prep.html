계정 3종 만들기 — 강사 노트

수강생용 준비 페이지 ↗ 를 사전 과제로 배포하는 것이 핵심. 여기서는 사전 과제 운영과 자주 막히는 지점만 다룹니다.

🎙️ 사전 안내 문자/메일 멘트

“강의 첫날을 알차게 쓰기 위해, 계정 3개(GitHub · Streamlit · AI)를 미리 만들어 오세요. 링크와 방법은 이 페이지에 있습니다: [수강생용 준비 URL]. 10분이면 됩니다. 가입 이메일·비번은 꼭 메모해 오세요.”

🎯 사전 과제로 내면 얻는 것

  • 1일차 30분 이상 절약 — 계정 세팅을 실습 시간에서 뺀다.
  • 못 해온 사람만 개강 직후 10~15분 집중 지원(조교 소회의실 지원 + 조원끼리).
📖 왜 세 계정인가 (역할 분리)

GitHub=코드 창고+정적 배포, Streamlit=코드가 실행되는 웹앱으로, AI=코드 생성기. 이 삼각형이 강의 전체를 관통합니다. 수강생에겐 “창고·집·요리사”로 비유.

🧯 계정별 자주 막히는 지점

① GitHub

🧯 예상 오류

Username을 아무렇게나 지음 → 나중에 사이트 주소에 그대로 노출. 개강 전 “짧고 깔끔한 영어”로 안내. · 이메일 인증 메일이 안 옴 → 스팸함 확인, 회사메일이면 차단되는 경우 있어 개인 Gmail 권장.

🙋 예상 질문

“Public이면 아무나 보나요?” → 코드는 볼 수 있지만 수정은 못 함. 우리가 올릴 건 공개돼도 괜찮은 것들이라 안전. (비밀 키는 3일차에 따로 배움.)

② Streamlit Community Cloud

🧯 예상 오류

GitHub 아닌 다른 방식으로 로그인 → 저장소 연결이 안 됨. 반드시 “Continue with GitHub”. · Authorize를 건너뜀 → 배포 시 저장소가 안 보임. 다시 로그인해 권한 허용.

③ AI 서비스

🎙️ 멘트 — 도구 중립 강조

“강사 화면은 Claude로 보여드리지만, ChatGPT·Gemini 무엇을 써도 이 강의 프롬프트는 똑같이 작동해요. 익숙한 것 하나만 준비하세요.”

🙋 예상 질문

“유료 결제해야 하나요?” → 무료로 충분합니다. 무료 한도를 넘으면 잠깐 기다리거나 다음날 이어서. 결제 정보는 넣지 않도록 안내.

⏱️ 개강 직후 5분 점검 루틴

확인손드는 사람이 있으면
“GitHub 로그인 되시는 분?”조교가 소회의실·채팅으로 미로그인자 지원
“Streamlit에 GitHub로 들어가지신 분?”Authorize 재실행 안내
