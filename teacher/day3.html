API와 AI 채팅앱 — 강사 진행안

수강생용 3일차 페이지 ↗ 를 화면에 띄운 채 진행합니다. 오늘의 산봉우리는 "처음 보는 API를 혼자 붙이는 5단계"의 체득입니다. 앱 완성은 그 부산물이고요. KOBIS는 전원이 직접 가입·발급하고, 유튜브·Solar 키는 강사가 배부합니다. 여기에는 시간표 · 멘트 · 예상 질문 · 예상 오류 · 이론 심화를 덧댔습니다.

🎙️ 오늘의 목표 선언 (시작 멘트)

“어제까지는 파일에 담긴 데이터를 열었어요. 오늘은 지금 이 순간의 데이터를 인터넷에서 직접 받아옵니다. 그런데 오늘은 순서가 조금 특별해요. 앞의 절반 동안은 AI에게 코드를 시키지 않습니다. 먼저 API가 뭔지 맨눈으로 보고, 그다음에 AI를 부릅니다. 그래야 다음에 처음 보는 API를 만나도 혼자 붙일 수 있거든요.”

📖 이번 개편의 설계 의도 — 왜 프롬프트를 미루나

이전 구성은 첫 실습부터 완성 코드가 나와서, API가 블랙박스인 채로 앱만 완성되는 문제가 있었습니다. 새 구성은 KOBIS 하나로 5단계(문서 확인 → 키 발급 → 원본 확인 → 값 꺼내기 → 바이브 코딩)을 천천히 오르고, 그다음 유튜브·AI에 같은 걸음을 반복합니다. 3·4절의 "함께 읽는 코드"는 AI가 만든 게 아니라 교재가 준비한 코드입니다 — "이건 제가 드리는 코드예요, AI는 아직입니다"라고 명확히 말해 주세요. 3절(글자 덩어리)과 4절(대시보드)의 화면 대비가 오늘의 정서적 절정이니, 두 화면을 나란히 보여주며 "같은 데이터입니다"를 꼭 말로 찍어 주세요.

1~5절이 오늘의 몸통입니다. 여기서 밀리면 6~8절을 줄이지, 1~5절을 서두르지 마세요. 6절(유튜브)은 두 번째 창구라 실제로 빨리 끝나고, 7절 회사별 탭은 "교재에 있으니 필요한 분만"으로 넘기고, 8절은 최악의 경우 강사 시연으로 대체해도 됩니다(프롬프트가 교재에 있으니 집에서 재현 가능). 무른 경계 원칙은 그대로 — 어떤 블록도 하드 데드라인이 아닙니다.

블록별 진행 노트

0. 복습 + 5단계 지도 (0~10′)

🎙️ 멘트 — 전환 훅

“어제 만든 그래프는 파일에 저장된 과거의 숫자였어요. 오늘 받아올 박스오피스 순위는 어제 극장의 숫자, 유튜브 구독자는 지금 이 순간의 숫자예요. 그리고 오늘의 진짜 목표는 이겁니다 — 이 수업이 끝나면 여러분은 처음 보는 API를 혼자 붙일 수 있는 순서를 갖게 됩니다.” 수강생 교재의 🗺️ 오늘의 지도(5단계)를 화면에 띄우고, 오늘 하루 종일 이 지도로 되돌아온다고 예고.

1. API와 JSON (10~30′)

🎙️ 멘트 — 민원창구 비유

“API는 관공서 민원창구예요. 정해진 창구에 정해진 서류(요청)를 내면, 정해진 양식(응답)으로 답을 줍니다. API 키는 신분증이고요.” 칠판에 ‘요청 → [창구] → 응답(JSON)’ 화살표를 그려 두고 오늘 내내 가리키기. 이어서 KOBIS 실제 응답을 띄우고 상자 속 상자를 손가락으로 따라가기: boxOfficeResult → dailyBoxOfficeList → 영화 한 편.

📖 이론 심화 — 따옴표 숫자, 왜 생기나

JSON 자체는 숫자 타입이 있지만, KOBIS는 모든 값을 문자열로 내려줍니다(공공 API에 흔한 방식). "820653"은 글자라 정렬하면 사전순("9"가 "10000"보다 큼)이 되는 게 4절 함정의 정체. 지금은 "따옴표에 싸여 있죠? 4절에서 이게 사고를 칩니다"라고 예고만 — 실제 해결은 4절 코드의 숫자 변환 넉 줄에서. 상태코드는 셋만: 200(성공)·404(없음)·429(너무 많이).

2. KOBIS 가입·키 발급 (30~50′)

🎙️ 멘트 — 전원 발급, 같이 클릭

“오늘은 열쇠를 여러분이 직접 만듭니다. 무료고, 신청하면 바로 나와요. 이 경험이 있으면 다음에 어떤 API를 만나도 겁이 안 납니다.” 강사 화면을 크게 띄우고 가입 → [키 발급/관리] → 발급 → 복사를 같이 클릭. 발급 후 바로 문서로 이동해 세 곳(요청 주소·요청변수·응답 예시)을 같이 찾고, “이 세 곳을 복사해 두세요, 5절에서 AI에게 줄 재료입니다”라고 예고.

🧯 예상 오류

인증 메일이 안 옴 → 스팸함 확인 · 그래도 안 오면 휴식 시간에 개별 지원. 가입이 계속 막히는 분 → 예비 키를 준비해 두었다가 조용히 전달하고 진도는 같이 (발급 경험은 짝의 화면으로 보게). 키 복사 실수 → 앞뒤 공백·일부 누락이 3절 faultInfo의 주범이니 "전체 선택 후 복사"를 강조.

🙋 예상 질문

“문서가 너무 길고 어려워요.” → "다 안 읽습니다. 주소·요청변수·응답 예시 세 곳만 찾으면 끝이에요. 나머지는 세부 옵션." · “이 키도 돈 나가요?” → KOBIS는 완전 무료(일 3,000회 한도, 수업엔 넉넉).

3. 프롬프트 없이 첫 요청 (60~80′)

🎙️ 멘트 — 주소창의 탈신비화

“API 요청, 대단한 게 아니에요. 주소를 여는 것입니다.” 교재의 🔗 주소 만들기 상자에 키만 붙여넣게 하세요(어제 날짜는 자동) → 복사한 주소를 주소창에 넣어 엔터 → 글자 덩어리가 뜨면 “여러분, 방금 API를 호출하셨어요. 축하합니다.” 주소를 손으로 편집시키지 말 것 — 한 글자 실수가 이 절 최대 병목입니다. 그다음 일부러 키를 한 글자 지우고 다시 열어 faultInfo를 보여 주세요 — “번호표(상태코드)는 200인데 서류 내용이 오류죠? 그래서 내용까지 확인해야 합니다.” 이어서 함께 읽는 코드를 한 줄씩 소리 내어 같이 읽기 — 금고에서 열쇠, 어제 날짜(한국 시간 기준), 요청, 보여주기. 이게 전부라는 것.

📖 이론 심화 — secrets, 이름 세 곳 일치

오늘 가장 공들일 개념입니다. 금고의 원리 = 값은 코드 밖에, 코드는 이름으로만 참조. 그래서 ①금고 라벨(KOBIS_KEY = "…") ②코드(st.secrets["KOBIS_KEY"]) ③프롬프트("비밀 금고의 KOBIS_KEY에서")의 이름 세 곳이 한 글자도 다르면 안 됩니다. 수강생 교재의 표를 띄우고 "바이브 코딩할 때도 ③처럼 이름을 콕 집어 말해야 AI가 같은 이름으로 코드를 씁니다"까지 연결. 금고가 두 곳(로컬 파일 vs 클라우드 Settings→Secrets)이라는 것도 여기서.

🧯 예상 오류

KeyError: 'KOBIS_KEY' → 이름 세 곳 불일치(소문자·하이픈·오타) 또는 클라우드 Secrets에 안 넣음. 오늘 최다 빈출이니 표로 돌아가 하나씩 대조. · faultInfo → 키 값 문제(공백·누락). · TOML 오류 → 따옴표 누락(KOBIS_KEY = 키값처럼 따옴표 없이 쓴 경우). · 데이터가 빈 목록 → targetDt에 오늘 날짜(아직 집계 전) 또는 여덟 자리 형식 위반.

4. JSON → 대시보드 (80~95′)

🎙️ 멘트 — 오늘의 절정

코드를 통째로 바꿔치기하고 앱이 다시 뜨는 순간: “3절 화면 기억나시죠? 글자 덩어리. 지금 이 카드와 그래프, 같은 데이터입니다. 코드가 한 일은 상자에서 값을 꺼내 자리에 놓은 것뿐이에요. 여러분이 매일 쓰는 화려한 앱들도 속은 다 이겁니다.” 시간이 되면 숫자 변환 넉 줄을 주석 처리해 그래프가 엉키는 것을 라이브로 — 1절에서 예고한 함정의 회수.

🙋 예상 질문

“이 코드를 외워야 하나요?” → 아니요. 읽을 수 있으면 충분합니다. 쓰는 건 다음 절부터 AI가 합니다. · “왜 아직 AI를 안 써요?” → "지금 여러분은 AI가 줄 코드를 읽고 판단할 눈을 만드는 중이에요. 5분 뒤에 바로 씁니다."

5. 문서 붙여 바이브 코딩 (95~120′)

🎙️ 멘트 — 오늘의 요령 선언

“이제 AI를 부릅니다. 오늘의 요령은 하나 — API 문서를 통째로 복사해 프롬프트 끝에 붙이는 것. 2절에서 복사해 둔 세 곳, 지금 씁니다. 문서를 주면 AI가 필드 이름을 지어내지 않아요.” ① 프롬프트의 구분선 아래(문서 붙인 부분)를 가리키며 구조를 설명. 완성 기준은 표와 그래프에 진짜 어제 영화가 찬다 — “오늘 저녁에 뉴스 검색해서 맞는지 확인해 보세요”로 비판적 사고 연결.

🧯 예상 오류

빈 화면 → ①Secrets 누락 ②오늘 날짜 조회(어제 자동 계산 줄이 빠짐). · 정렬·그래프가 이상함 → 숫자 변환 누락 — "①에 '숫자로 바꿔서'가 있는지 확인하고 다시 주문". · AI가 다른 창구(주간 박스오피스 등)를 씀 → 문서 붙인 부분이 빠졌는지 확인. 에러·화면 상태를 통째로 AI에게 붙여넣는 습관을 계속 강조.

📖 이론 심화 — 왜 '문서 붙이기'가 통하나

AI는 유명 API(유튜브 등)는 학습으로 알지만, 국내 공공 API의 필드명은 틀리게 기억하거나 지어내기 쉽습니다. 문서를 붙이면 AI가 기억 대신 눈앞의 사실로 코드를 쓰게 됩니다. 이 요령은 사내 API·신규 API 등 AI가 모르는 창구일수록 강력 — 5일차 프로젝트에서 공공데이터포털 API를 쓸 때 그대로 재사용됩니다.

6. 유튜브 채널 대시보드 (130~145′)

🎙️ 멘트 — 두 번째 창구, 같은 5단계

“조금 전에 오른 계단을 다른 창구에서 한 번 더 — 이번엔 유튜브입니다. 다만 유튜브 열쇠는 구글 클라우드에서 만들어야 해서 단계가 많아요. 오늘은 제가 나눠 드리고, 직접 만드는 법은 교재 참고 자료에 있습니다.” 키를 채팅으로 배부 → 금고에 추가(줄 늘리기) → ① 프롬프트. 두 번째라 진도가 빠르니, 남는 시간은 LV4 개선 놀이로.

🧯 키 배부 운영 + 예상 오류

배부 요령 — 유튜브·Solar 모두 실습용 키를 여러 개 준비해 조별로 나눠 배부(한 키에 40명이 몰리면 할당량·429 위험) · Zoom 채팅으로 전달 · 수업 후 키 삭제·재발급으로 회수. 빈 화면 → 1순위 채널 ID vs 핸들 혼동(@handle은 ID 아님 · ID는 UC…), 2순위 Secrets 누락. 403 → 집에서 직접 발급한 키의 사용 설정(Enable) 누락이 대부분.

🙋 예상 질문

“구독자 수가 실제랑 조금 달라요.” → 유튜브는 큰 수를 반올림해 제공. 정상. · “KOBIS처럼 브라우저로도 돼요?” → 됩니다! 여유 있는 분께 channels 주소에 key를 붙여 열어 보게 하면 "5단계가 정말 똑같구나"가 완성됩니다.

7. 나만의 AI 채팅앱 (145~160′)

🎙️ 멘트 — 데이터 API와 다른 점

“유튜브 API는 물어보면 늘 같은 답이었죠? 이번 AI는 매번 새로 문장을 만들어 냅니다. 같은 질문을 두 번 해서 답이 다른 걸 직접 보여 주세요. 그래도 순서는 똑같아요 — 주소, 열쇠, 요청, 응답.” Solar 키 배부 → 금고 세 번째 줄 → ① 프롬프트. "우리나라 회사(업스테이지)가 만든 AI를 무료로 부른다"는 점도 한마디. 회사별 탭은 펼쳐서 30초만 — “네 회사 모두 같은 라이브러리, 바뀌는 건 주소·모델·열쇠 셋뿐. 원하는 분은 집에서 탭대로.”

📖 이론 심화 — 회사별 탭의 검증 등급

교재의 전 과정 실습 검증은 Solar(solar-open2) 기준입니다. Gemini·ChatGPT·Claude 탭은 발급 경로와 "바꿔 말할 부분" 안내이며, 모델 이름은 회사가 수시로 바꾸므로 학기마다 점검 대상(오류 시 "현재 저렴한 모델 이름을 AI에게 묻기"로 안내). 개념 셋만 챙기면 충분: 토큰(요금·길이 단위) · 무료 한도(초과 시 429) · 웹 구독과 API 크레딧은 별개(가장 흔한 오해).

🧯 예상 오류

답이 안 오고 한참 조용 → reasoning_effort="none"(추론 끄기) 누락 — 추론 모델이라 이 옵션이 없으면 수십 초 생각부터 함(실측: none 0.7초 vs 기본 9초). · 글자가 안 흐르고 한꺼번에 뜸 → ①의 스트리밍 줄이 빠진 것, 동작엔 문제 없음. · 모델명 오타 → solar-open2 글자 그대로(접속 주소 https://api.upstage.ai/v1). · 대화가 기억 안 됨 → "대화가 이어지게 세션에 저장해줘"로 재주문. · 429 → 무료 한도, 잠시 후. · 키 하드코딩(재경고) → AI 키도 반드시 금고에.

🎙️ 멘트 — 과금 폭탄 예방 (교재 💸 박스)

교재의 💸 박스를 띄우고 세 줄로: “①키 유출이 과금 폭탄 1순위 — secrets 습관이 곧 지갑을 지킵니다. ②좋은 모델일수록 비쌉니다 — 연습은 무료·저렴한 모델로 충분. ③유료 결제 시 콘솔에서 월 사용 한도부터 설정.”

8. 유튜브 댓글 분석 (160~172′)

🎙️ 멘트 — 두 세계의 결합

“마지막 앱은 세 단계로 키워 갑니다 — 1단계 링크 붙여넣어 댓글 불러오기, 2단계 단어 빈도 그래프, 3단계 워드클라우드. 단계마다 '이어서 고쳐줘'로 프롬프트를 쌓아요. 이 리듬이 5일차 프로젝트의 리듬입니다.” 입력창에 기본값 링크(딥마인드 다큐)가 미리 들어 있어 버튼만 눌러도 시연 가능 · 한글 워드클라우드를 보여줄 땐 예시 2(2002 월드컵 추억, 14F) 버튼으로 전환. 단계마다 ✔️ 확인을 짧게 밟고 다음으로. 한글 워드클라우드가 뜨는 순간이 절정 — “댓글 100개가 그림 한 장이 됐어요”를 같이 감탄. 완성 기준은 3단계 워드클라우드, 최소 기준은 1단계 댓글 불러오기 · ③ AI 요약은 선택.

🧯 예상 오류

링크에서 ID를 못 찾음 → 영상 링크가 아니라 채널·재생목록 링크를 넣은 경우가 대부분 · 공유 버튼으로 복사한 영상 주소를 쓰게 안내. · 댓글이 안 옴 → 댓글 막힌 영상이면 다른 영상으로 · 기본값 링크로 되돌리면 바로 확인 가능. · 글자가 네모(□□)로 깨짐 → 한글 폰트 누락. 3단계 ①의 폰트 주소 줄이 빠졌는지 확인. · 2단계 그래프가 이상함 → 단어 나누기가 공백 기준이라 조사가 붙은 채 세어짐 — 오류 아님, ③ 개선("의미 없는 말은 빼줘")으로 다듬는 소재로. · wordcloud 설치 오류 → requirements.txt에 wordcloud 누락. · ③ 요약이 느림·429 → 댓글 100개면 요청이 길어 몇 초 걸림, 버튼 연타 금지.

🙋 예상 질문

“댓글이 100개보다 많은데요?” → 오늘은 창구가 한 번에 주는 최대 100개만. 더 받으려면 "다음 페이지" 개념이 필요한데, 그건 AI에게 "댓글을 300개까지 받아와줘"라고 시키면 알아서 해 줍니다. · “그래프에 '너무', '정말' 같은 말만 커요.” → 좋은 발견! ③ 개선의 불용어 입력창에 적으면 그래프·워드클라우드에서 빠집니다. · “영어 댓글도 섞여 있어요.” → 정상. 전 세계 시청자의 댓글이라 그렇고, 워드클라우드에 함께 그려집니다.

9. 혼잡도 시연 + 갤러리 + 숙제 (172~180′)

🎙️ 멘트 — 프로젝트 동기부여 시연

“오늘 배운 것들이 모이면 어디까지 가는지 보여 드릴게요.” 서울 혼잡도 애니메이션 지도의 재생 버튼을 눌러, 새벽엔 점이 쪼그라들고 낮·저녁에 부푸는 걸 감상. “API 한 번은 스냅샷인데, 시각마다 모아 두면 이렇게 하루가 영상이 됩니다. 5일차 자유 프로젝트에서 여러분 주제로 이런 걸 만들 수 있어요.” 수업 그 시각의 실제 붐빔과 대조하면 반응이 좋습니다. 이어서 갤러리 게시(5분) — 오늘 만든 앱 중 하나 이상 올리고 서로 구경.

🧯 시연 팁 · 함정

강사 시연 3~5분이면 충분(수강생은 교재의 프롬프트로 집에서 재현 가능 · 단 무료 하한급 AI는 지도 코드에서 자주 실패하니 상위 모델 권장 — 강건성 실측 결과). · 데이터는 seoul_congestion_log.csv(예제 저장소) — 별도 키 불필요. · ‘약간 붐빔’에 띄어쓰기 포함(전처리 주의). · 명소 23곳은 통계 아닌 관찰 → "23곳 중에선"으로만 말하기(일반화 금지). · 저녁 수업이면 대부분 ‘여유’라 밋밋 → 낮 피크 스냅샷을 미리 캡처해 대조군으로. · 로그에 10~11시 프레임이 비어 있음(수집 공백) — 슬라이더가 9시에서 12시로 건너뛰는 건 오류가 아니라 "수집이 쉬었던 시간"이라고 안내.

🎙️ 마무리 멘트 — 숙제와 내일 준비

“오늘 배운 5단계는 세상 거의 모든 API에 통합니다. 숙제는 둘이에요. ①부록의 키 없는 API 하나로 5단계를 혼자 한 바퀴(키가 없으니 2단계는 건너뛰고). ②공공데이터포털과 서울 열린데이터광장을 구경하며 내 관심 주제의 API를 하나 찾아 문서 세 곳만 훑어 두기 — 만들 필요 없어요, 5일차 프로젝트의 재료입니다. 그리고 내일 준비로 Supabase 가입 5분(교재 끝 🌙 안내), 선택이지만 강력 추천.”
