API와 AI 채팅앱 — 강사 진행안

수강생용 3일차 페이지 ↗ 를 화면에 띄운 채 진행합니다. 키는 강사가 배부(유튜브·Solar 모두)하므로 발급은 수업에서 하지 않습니다. 오늘의 산봉우리는 API 개념 첫 왕복과 키 보안(secrets) 습관입니다. 여기에는 시간표 · 멘트 · 예상 질문 · 예상 오류 · 이론 심화를 덧댔습니다.

🎙️ 오늘의 목표 선언 (시작 멘트)

“어제까지는 파일에 담긴 데이터를 열었어요. 오늘은 지금 이 순간의 데이터를 인터넷에서 직접 받아옵니다. 그리고 나만의 AI를 내 앱 안에 넣어요. 오늘 딱 하나만 몸에 새기면 됩니다 — 열쇠(키)는 코드에 쓰지 않는다.”

📖 교재의 핵심 장치 — ①②③ 세 박자 유지

3일차도 모든 만들기가 ① 샘플 프롬프트(청록) → ② AI가 준 코드 예시(갈색) → ③ 프롬프트 개선 예시(주황) 리듬입니다. 오늘은 ②(예시 코드)에 st.secrets 가 등장하는 게 핵심입니다. “키 값이 코드 어디에도 없죠?”를 반드시 강조하세요. 그게 오늘의 안전벨트입니다.

블록별 진행 노트

0. 복습 (0~15′)

🎙️ 멘트 — 전환 훅

“어제 만든 그래프, 그거 파일에 저장된 과거의 숫자였어요. 오늘 우리가 받아올 유튜브 구독자 수는 지금 이 순간의 숫자예요. 새로고침하면 또 달라요. 그래서 오늘은 이런 ‘살아있는(실시간) 데이터’를 다뤄요.”

📖 이론 심화 — 정적 파일 vs API

2일차 CSV는 내려받은 스냅샷(누가 언제 만든 고정 데이터). API는 매 요청마다 서버가 최신값을 응답. 이 대비를 복습에서 심어야 “왜 API를 배우나”가 열립니다. 어제의 ‘비판적 사고’는 오늘도 유효 — API 응답도 결측·이상치가 있을 수 있음.

1. API 개념 + 키 없는 워밍업 (15~60′)

🎙️ 진행 — 45분을 확보한 이유

이 블록이 오늘의 사고 골격(요청 → 응답 → 값 꺼내기)입니다. 키 발급을 뺀 대신 여기에 시간을 몰았으니, 개념 15분 + 실습 30분으로 천천히. 수강생 교재에 ②(예시 코드)가 있으므로 "여러분 코드가 이 모습이면 성공"으로 안심시키고, ✔️ 확인(버튼마다 다른 상식)까지 전원 통과를 보고 넘어갑니다.

🎙️ 멘트 — 민원창구 비유

“API는 관공서 민원창구예요. 정해진 창구에 정해진 서류(요청)를 내면, 정해진 양식(응답)으로 답을 줍니다. API 키는 신분증이고요.” — 칠판에 ‘요청 → [창구] → 응답(JSON)’ 화살표를 그려 두고 오늘 내내 가리키기.

📖 이론 심화 — 상태코드와 JSON

상태코드 셋만: 200(성공)·404(없음)·429(너무 많이). JSON은 {"라벨": 값} 구조이며 data["fact"] 처럼 라벨로 값을 꺼낸다는 것만 이해시키면 충분. 키 없는 catfact.ninja·randomuser.me 로 ‘요청 → 응답 → 값 꺼내기’ 전 과정을 부담 없이 한 바퀴 돌리는 게 목표. st.expander 로 원본 JSON을 펼쳐 “라벨이 이렇게 생겼구나”를 눈으로.

🧯 예상 오류

ModuleNotFoundError: requests → 스트림릿 클라우드 배포 시 requirements.txt 에 requests 추가(로컬 실행이면 이미 있을 수 있음). · KeyError → JSON 라벨 이름 오타. 응답을 st.write(res.json()) 로 한번 펼쳐 실제 라벨 확인. · 429 → 버튼 연타. “잠깐 쉬었다 다시”가 곧 429의 뜻임을 실시간으로 체험시키기 좋은 순간.

2. 유튜브 채널 대시보드 (70~115′)

🎙️ 멘트 — 열쇠는 나눠 주고, 습관에 집중

“열쇠(API 키)는 오늘 제가 나눠 드려요. 지금 채팅으로 보낸 키를 복사하세요. 대신 오늘 배울 건 열쇠 만들기가 아니라 열쇠를 안전하게 쓰는 법입니다 — 금고(secrets)에 넣고, 코드에는 절대 쓰지 않기.” 이후 수강생 교재의 레벨업 계단(LV1 빈 앱 → LV2 금고 → LV3 카드 4개 → LV4 개선)을 따라 한 계단씩, 계단마다 성공 확인 후 다음으로. 직접 발급해 보고 싶다는 분께는 교재의 (참고) 집에서 내 키 만들기를 안내.

🧯 예상 오류 (오늘 최대 위험) — 키 하드코딩 → GitHub 노출

가장 강력히 경고할 지점. 수강생이 API_KEY = "AIza..." 처럼 코드에 키를 직접 적고 GitHub에 올리면 키가 전 세계에 공개됩니다(봇이 몇 분 만에 긁어 감 → 요금 폭탄·계정 정지 위험). 반드시 ①.streamlit/secrets.toml 에만 적고 코드는 st.secrets 로 읽기 ②.gitignore 에 .streamlit/secrets.toml 추가 — 이 두 가지를 배포 전에 확인시키세요. “키가 코드에 보이면 그건 이미 유출”이라고 각인.

🧯 예상 오류 (빈출)

빈 화면(에러 없음) → 1순위 채널 ID vs 핸들 혼동(@handle 은 ID 아님 · ID는 보통 UC… · 기본값 UC_x5XG1OV2P6uZZ5FSM9Ttw), 2순위 배포 앱 Settings→Secrets 누락. 수강생 교재의 ✔️ 확인 박스를 그대로 읽게 하기. · 429/할당량 초과 → 공용 실습 키라 여럿이 몰리면 발생 가능. 조별로 다른 키를 쓰게 하고, 소진되면 예비 키로 교체. · secrets 오타 → TOML 따옴표·키 이름(YOUTUBE_API_KEY) 불일치. · 403 / “API not enabled” → 집에서 직접 발급한 키에서 주로 발생(사용 설정 단계 건너뜀). 배부용 키는 강사가 미리 Enable·제한 없음 상태로 준비.

🙋 예상 질문

“영상당 평균 조회수는 어떻게 나와요?” → 총 조회수 ÷ 영상 수. AI가 계산해 st.metric 에 넣게 프롬프트에 명시돼 있음. · “구독자 수가 실제랑 조금 달라요.” → 유튜브는 큰 수를 반올림해 제공(예: 1,230,000). 정상.

3. 나만의 AI 채팅앱 (125~170′)

🎙️ 멘트 — 데이터 API와 다른 점

“유튜브 API는 물어보면 늘 같은 답이었죠? 이번 AI는 매번 새로 문장을 만들어 냅니다. 같은 질문을 두 번 해서 답이 다른 걸 직접 보여 주세요.” Solar 키도 강사가 배부(채팅 전달) — 받은 키를 secrets에 유튜브 키와 나란히 추가하는 것부터. 두 번째라 금고 넣기는 금방 끝나니, 남는 시간은 대화 체험과 성격(시스템 프롬프트) 놀이에 몰아 주세요. "우리나라 회사(업스테이지)가 만든 AI를 무료로 부른다"는 점도 한마디.

📖 이론 심화 — 데이터 API vs LLM API

핵심 대비: 데이터 API = 고정된 사실을 조회(결정적, 같은 입력→같은 출력) vs LLM API = 확률적으로 문장을 생성(같은 입력→매번 조금 다른 출력). 추가로 세 개념만: 토큰(글자를 잘게 나눈 요금·길이 단위, 무료 한도 안에서 놀기), 무료 한도(분당/일당 요청 제한 → 초과 시 429), 맥락(context) 주입(유튜브 댓글 등을 프롬프트에 함께 넣어 “이걸 요약해줘” — 데이터 API와 LLM API를 합치는 순간). session_state=대화 기억, 시스템 프롬프트=성격 부여로만 설명하면 충분.

🧯 예상 오류

키 하드코딩(다시 경고) → AI 키도 반드시 st.secrets. 유튜브와 똑같은 습관. · ModuleNotFoundError → requirements.txt 에 AI 라이브러리(openai) 추가. · 모델명 오타/구버전 → solar-open2 로 통일(접속 주소 https://api.upstage.ai/v1 확인). · 답이 안 나오고 한참 조용 → reasoning_effort="low" 빠졌는지 확인(추론 모델이라 이 옵션이 없으면 수십 초 생각부터 함 · 수강생 교재 ✔️ 박스에 "조용해도 고장 아님" 안내 있음). · 대화가 기억 안 됨 → session_state 초기화 위치 문제. AI에게 “대화가 이어지게 session_state에 저장해줘”로 재요청. · 429 → 무료 한도 초과, 잠시 후.

🙋 예상 질문

“돈 나가요?” → 오늘 쓰는 건 무료 한도 안. 유튜브·Solar 모두 개인 실습 수준은 무료 범위(Solar는 베타 기간 무료). 다만 키를 남이 주워 쓰면 그 사람이 내 한도를 태울 수 있으니 보안이 곧 절약. · “키가 남한테 보이면 어떻게 돼요?” → 남이 내 신분증으로 서비스를 씀 → 한도 소진·요금·정지 위험. 그래서 secrets에 보관 + 저장소는 비공개(또는 .gitignore). 이미 노출됐다면 콘솔에서 키 삭제 후 재발급이 정답.

🎙️ 멘트 — 과금 폭탄 예방 3가지 (교재 💸 박스)

수강생 교재의 “💸 과금 폭탄 주의” 박스를 화면에 띄우고 세 줄로 정리해 주세요: “①키 유출이 과금 폭탄의 1순위예요 — 오늘 배운 secrets 습관이 곧 지갑을 지킵니다. ②좋은 모델일수록 비쌉니다 — 연습은 flash·mini·Haiku 같은 빠르고 저렴한 모델로 충분해요. 최고급 모델은 수십 배 요금. ③유료 결제를 시작하면 콘솔에서 월 사용 한도부터 설정 — 실수해도 거기서 멈춥니다.”

📖 참고 — 챗지피티·클로드 키 (교재 🗝️ 표)

수강생 교재에 Gemini(aistudio.google.com)·ChatGPT(platform.openai.com)·Claude(platform.claude.com) 발급 안내 표가 있습니다. 수업에선 Solar만 실습하고 표는 “다른 회사도 방법은 비슷하다” 수준으로만 소개. 확인할 핵심 둘: ①Gemini는 무료 한도가 있지만 ChatGPT·Claude는 선불 크레딧(카드 등록/충전)이 필요 — Solar와 달리 무료 한도가 사실상 없음. ②웹사이트 구독(ChatGPT Plus 등)과 API 크레딧은 별개 — 구독 중이어도 API는 따로 충전해야 함(가장 흔한 오해). 키가 발급 순간 한 번만 보인다는 것도 공통.

4. 실시간 API → 애니메이션 (하이라이트 데모 · 시간 되면)

🎙️ 멘트 — 스냅샷을 쌓으면 영상이 된다

“API 한 번 = 지금 이 순간이에요. 서울 실시간 혼잡도 API를 매시간 자동으로 쌓아 두면 시간축이 생겨 하루 애니메이션이 됩니다.” 수강생 §4의 재생 버튼을 눌러 새벽엔 점이 쪼그라들고 낮·저녁에 부푸는 걸 보여 주세요. 여유되면 2일차 거주 고령 지도(면)를 배경으로 깔아 “밤에 사는 사람 vs 낮에 있는 사람” 대비까지(가산·강남=낮에 폭발 / 고덕·구로=낮밤 비슷).

🧯 진행 팁 · 함정

완성 목표 아님 — 강사 화면공유 데모 3~5분이면 충분(수강생은 여유 시 프롬프트 복사). · 데이터는 seoul_congestion_log.csv(예제 저장소, 매시간 append) — 별도 키 불필요. · ‘약간 붐빔’에 띄어쓰기 포함(전처리 주의). · n=23은 통계 아닌 관찰 → “23곳 중에선”으로만 말하도록(일반화 금지). · 저녁 수업이면 대부분 ‘여유’라 밋밋 → 낮 피크 스냅샷을 미리 캡처해 대조군으로. · 이건 10장(자동수집)·2일차(지도)의 합류점이라 “오늘 배운 게 다 모인다”로 프레이밍.
