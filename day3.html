살아있는 데이터: API와 나만의 AI

오늘의 약속 — 남이 가진 따끈따끈한 데이터를 요청해서 받아옵니다. 유튜브 채널 대시보드와 나만의 AI 채팅앱을 만들어 집에 갑니다.

🎯 오늘 만드는 것
  • 유튜브 채널 대시보드 — 구독자·조회수를 API로 불러와 카드로
  • 나만의 AI 채팅앱 — 말을 걸면 대답하는 AI를 내 앱 안에

0. 지난 시간 잠깐 복습

2일차에 우리는 진짜 데이터를 다뤘어요. 딱 네 가지만 떠올려 보세요.

  • pandas — 표(엑셀 같은 데이터)를 다루는 도구. df 라는 이름으로 자주 만났죠.
  • Plotly — 숫자를 예쁜 그래프로 그려 주는 도구.
  • 지도 — 위도·경도가 있으면 데이터를 지도 위에 점으로 찍을 수 있었어요.
  • 의심하는 눈 — 그래프가 나오면 “진짜야?”, “축이 이상하지 않아?” 하고 한 번 더 보는 습관.

오늘은 여기서 한 걸음 더 나갑니다. 지금까지는 파일에 담긴 데이터를 열었다면, 오늘은 지금 이 순간의 데이터를 인터넷에서 직접 받아옵니다.

1. API란 무엇일까 — 관공서 민원창구

API는 어렵게 생각할 필요 없어요. 관공서 민원창구를 떠올리면 됩니다.

주민센터에 가면 정해진 창구가 있죠. 정해진 서류(요청)를 내면, 직원이 정해진 양식(응답)으로 답을 줍니다. API도 똑같아요. 우리가 유튜브·날씨·AI 회사의 정해진 창구(주소)에 “이거 알려줘” 하고 요청을 보내면, 정해진 양식으로 응답이 돌아옵니다. 그 양식이 보통 JSON이라는 형식이에요.

민원창구API설명
창구 주소요청 URL어디에 물어볼지 정해진 인터넷 주소
제출 서류요청(request)“무엇을 알려줘”라는 우리의 물음
돌려받는 서류응답(response)정해진 양식(JSON)으로 오는 답
신분증API 키“나 이 서비스 쓰는 사람이에요”를 증명하는 열쇠

답이 올 때는 상태코드라는 번호표도 함께 옵니다. 딱 셋만 알아두세요.

  • 200 — 성공! 잘 받았어요.
  • 404 — 없어요. 주소가 틀렸거나 찾는 게 없을 때.
  • 429 — 너무 많이 물어봤어요. 잠깐 쉬었다 다시.

키 없이 몸풀기 — 고양이 상식과 랜덤 유저

본격적으로 들어가기 전에, 신분증(키)이 필요 없는 창구로 연습해 봐요. 스트림릿 앱에서 requests라는 도구로 창구에 요청을 보내고, 돌아온 JSON에서 원하는 값을 꺼내 봅니다.

① 샘플 프롬프트
스트림릿(streamlit) 앱 하나(main.py)를 만들어줘. API 첫 연습용이야.

- 버튼을 누르면 https://catfact.ninja/fact 주소에 requests로 요청을 보내고,
  돌아온 JSON에서 'fact' 값(고양이 상식)을 꺼내 화면에 크게 보여줘.
- 아래에는 https://randomuser.me/api/ 로 요청해서 랜덤 인물 한 명의
  이름과 사는 나라를 st.write로 보여주는 버튼도 하나 넣어줘.
- 요청이 성공(상태코드 200)인지 확인하고, 실패하면 친절한 한국어 메시지를 보여줘.
- 초보자용으로 주석을 한국어로 달고, main.py 전체 코드를 한 번에 줘.

버튼을 눌러 보면, 누를 때마다 다른 고양이 상식과 다른 사람이 나와요. 파일에 저장된 게 아니라 그 순간 창구에서 받아온 값이라 그렇습니다. 이게 “살아있는 데이터”예요.

③ 프롬프트 개선 (예시)
좋아! 여기서 이어서 고쳐줘.
- 받아온 JSON 원본도 접었다 펼 수 있게(st.expander) 보여줘. 어떻게 생겼는지 궁금해.
- 고양이 상식을 한국어로 번역해서 함께 보여주면 더 좋겠어.
- 버튼 색과 카드를 따뜻한 노란 톤으로 예쁘게 꾸며줘.
💡 JSON은 ‘라벨 붙은 상자’예요

JSON은 {"이름": "값"} 처럼 라벨과 값이 짝지어진 형태예요. 그래서 data["fact"] 라고 하면 “data라는 상자에서 fact 라벨이 붙은 값을 꺼내 줘”가 됩니다. 어떤 라벨이 있는지는 응답을 한번 펼쳐 보면 알 수 있어요.

2. 유튜브 채널 대시보드 만들기

이제 신분증(키)이 필요한 진짜 창구로 갑니다. YouTube Data API v3를 쓰면 어떤 채널의 구독자 수·총 조회수·영상 수를 받아올 수 있어요. 이걸로 채널 통계 대시보드를 만듭니다.

먼저 열쇠(API 키) 받기 — 3단계

구글 클라우드 콘솔(console.cloud.google.com)에서 키를 발급받아요. 처음이라 낯설지만, 딱 세 걸음이에요. 강사님 화면을 같이 보면서 천천히 따라오세요.

  1. 프로젝트 만들기
    상단의 프로젝트 선택 → 새 프로젝트. 이름은 my-youtube 처럼 아무거나. 만든 뒤 그 프로젝트를 선택해 둡니다.
  2. YouTube Data API v3 사용 설정
    왼쪽 메뉴 API 및 서비스 → 라이브러리 → YouTube Data API v3 검색 → 사용 설정(Enable).
  3. API 키 발급
    API 및 서비스 → 사용자 인증정보 → 사용자 인증정보 만들기 → API 키. 만들어진 키(긴 글자)를 복사해 둡니다.

🔒 이 열쇠, 코드에 그냥 쓰면 안 돼요!

API 키는 여러분의 신분증이에요. 이걸 코드에 그대로 적어서 GitHub에 올리면, 모르는 사람이 내 키를 주워 써 버릴 수 있어요. 그래서 키는 코드와 따로 보관합니다. 아래에서 안전한 방법을 배워요.

키를 안전하게 보관하기 — secrets

스트림릿에는 비밀 금고가 있어요. 프로젝트 안에 .streamlit/secrets.toml 이라는 파일을 만들고, 거기에만 키를 적습니다. 코드에서는 키 값을 직접 쓰지 않고 st.secrets 로 꺼내 씁니다.

.streamlit/secrets.toml
YOUTUBE_API_KEY = "여기에_발급받은_키를_붙여넣기"
📸[스크린샷] Streamlit 앱 Settings · Secrets 입력Settings의 Secrets 입력창에 YOUTUBE_API_KEY 줄이 채워진 모습이 보이게shot: day3-04-secrets
🧯 그리고 .gitignore 에 꼭 추가

이 비밀 파일이 실수로 GitHub에 올라가지 않도록, .gitignore 라는 파일에 .streamlit/secrets.toml 한 줄을 적어 둡니다. “이 파일은 GitHub에 올리지 마”라는 뜻이에요. AI에게 “.gitignore에 secrets.toml 넣어줘”라고 부탁하면 됩니다.

대시보드 만들기

이제 앱을 주문해요. 채널을 넣으면 구독자·총조회수·영상수, 그리고 영상당 평균 조회수(총조회수 ÷ 영상수)를 st.metric 카드로 보여 주는 대시보드입니다.

① 샘플 프롬프트
스트림릿(streamlit) 앱 하나(main.py)를 만들어줘. 유튜브 채널 통계 대시보드야.

- 사용자가 유튜브 채널 ID(예: UC_x5XG1OV2P6uZZ5FSM9Ttw)를 입력하면,
  YouTube Data API v3의 channels 창구에 requests로 요청해서
  구독자 수, 총 조회수, 영상 수를 받아와.
- API 키는 코드에 절대 쓰지 말고 st.secrets["YOUTUBE_API_KEY"] 로 불러와.
- 상단에 st.metric 카드 4개로 보여줘: 구독자, 총 조회수, 영상 수,
  그리고 '영상당 평균 조회수'(총 조회수 ÷ 영상 수, 정수로 반올림).
- 상태코드가 200이 아니거나 채널을 못 찾으면 친절한 한국어 안내를 보여줘.
- 한국어 라벨, 따뜻한 톤, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.

잠깐 기다리면 아래처럼 완성된 코드 한 덩어리가 옵니다. 여러분 것과 똑같지 않아도 괜찮아요 — 이런 모습이면 성공이에요.

② AI가 준 코드 (예시)
import streamlit as st
import requests

st.title("📺 유튜브 채널 대시보드")

# 🔑 키는 코드에 직접 쓰지 않고, 비밀 금고(secrets)에서 꺼냅니다
API_KEY = st.secrets["YOUTUBE_API_KEY"]

# 채널 ID 입력창 (기본값은 구글 개발자 채널)
channel_id = st.text_input("채널 ID", "UC_x5XG1OV2P6uZZ5FSM9Ttw")

if channel_id:
    # 유튜브 '창구' 주소에 요청을 보냅니다
    url = "https://www.googleapis.com/youtube/v3/channels"
    params = {"part": "statistics", "id": channel_id, "key": API_KEY}
    res = requests.get(url, params=params)

    if res.status_code == 200 and res.json().get("items"):
        # 응답(JSON) 상자에서 statistics 값을 꺼냅니다
        stats = res.json()["items"][0]["statistics"]
        subs = int(stats["subscriberCount"])
        views = int(stats["viewCount"])
        videos = int(stats["videoCount"])
        avg = round(views / videos) if videos else 0

        # metric 카드 4개로 보여주기
        c1, c2, c3, c4 = st.columns(4)
        c1.metric("구독자", f"{subs:,}")
        c2.metric("총 조회수", f"{views:,}")
        c3.metric("영상 수", f"{videos:,}")
        c4.metric("영상당 평균 조회수", f"{avg:,}")
    else:
        st.warning("채널을 찾지 못했어요. 채널 ID를 다시 확인해 주세요. 🙏")

결과를 보고 마음에 안 들면 대화로 고칩니다. 예를 들면 이렇게요:

③ 프롬프트 개선 (예시)
좋아! 여기서 이어서 고쳐줘.
- 채널 이름과 프로필 이미지도 함께 위에 크게 보여줘(snippet 정보 사용).
- 좋아하는 채널 3개를 미리 버튼으로 넣어서, 누르면 바로 통계가 뜨게.
- 영상당 평균 조회수가 100만이 넘으면 카드 옆에 🔥 이모지를 붙여줘.
💡 채널 ID는 어디서?

유튜브 채널 주소가 @채널핸들 형태라면 그건 핸들이지 채널 ID가 아니에요. 채널 ID는 보통 UC 로 시작하는 긴 글자예요. 못 찾겠으면 AI에게 “이 핸들의 채널 ID 찾는 법 알려줘”라고 물어보세요.

3. 나만의 AI 채팅앱 만들기

지금까지 만난 유튜브·고양이 API는 물어보면 정해진 답을 줬어요(같은 채널이면 늘 같은 통계). 그런데 AI(LLM) API는 달라요. 매번 새로운 답을 생성해 냅니다. 이걸 내 앱에 넣으면, 말을 걸면 대답하는 나만의 챗봇이 됩니다.

AI 열쇠 받기

여기서는 예로 구글의 Gemini를 씁니다. Google AI 스튜디오(aistudio.google.com)에서 Get API key를 눌러 키를 발급받아요. 모델은 빠르고 무료 한도가 넉넉한 gemini-2.5-flash 를 쓸게요. (원한다면 다른 회사의 AI를 써도 방식은 같아요.)

📸[스크린샷] AI 스튜디오 · API 키 발급Google AI 스튜디오의 Get API key 버튼과 새로 발급된 키가 보이게shot: day3-05-llm-key
🔒 이 키도 똑같이 secrets에

AI 키 역시 신분증이에요. 유튜브 키와 똑같이 .streamlit/secrets.toml 에만 적고, 코드에서는 st.secrets 로 꺼내 씁니다. 코드에 직접 쓰지 않기, GitHub에 올리지 않기 — 오늘의 가장 중요한 습관이에요.

.streamlit/secrets.toml (한 파일에 여러 키)
YOUTUBE_API_KEY = "유튜브_키"
GEMINI_API_KEY = "AI_스튜디오에서_받은_키"

대화가 이어지는 채팅앱

채팅앱은 세 가지 부품으로 만들어져요. st.chat_input(아래 입력창), st.chat_message(말풍선), session_state(지난 대화를 기억하는 메모). 그리고 시스템 프롬프트로 AI에게 성격을 부여할 수 있어요(“너는 친절한 데이터 선생님이야”처럼).

① 샘플 프롬프트
스트림릿(streamlit)으로 AI 채팅앱 하나(main.py)를 만들어줘.

- 구글 Gemini API(모델 gemini-2.5-flash)를 사용해.
- API 키는 코드에 쓰지 말고 st.secrets["GEMINI_API_KEY"] 로 불러와.
- st.chat_input 으로 사용자가 메시지를 입력하면, AI의 답을
  st.chat_message 말풍선으로 보여줘.
- session_state에 대화 기록을 저장해서, 이전 대화를 기억하며 이어서 답하게 해줘.
- 시스템 프롬프트로 '너는 따뜻하고 친절한 데이터 분석 선생님이야'라는 성격을 줘.
- 한국어로 대화하고, 초보자용 한국어 주석을 달아. main.py 전체 코드를 한 번에 줘.

아래는 받게 될 코드의 모습 예시예요. 라이브러리 이름이나 형태가 조금 달라도 괜찮아요.

② AI가 준 코드 (예시)
import streamlit as st
import google.generativeai as genai

st.title("🤖 나만의 AI 채팅")

# 🔑 AI 키도 비밀 금고에서 꺼냅니다
genai.configure(api_key=st.secrets["GEMINI_API_KEY"])

# 성격 부여(시스템 프롬프트) + 매번 새 답을 '생성'하는 모델
model = genai.GenerativeModel(
    "gemini-2.5-flash",
    system_instruction="너는 따뜻하고 친절한 데이터 분석 선생님이야."
)

# 지난 대화를 기억하는 메모장
if "messages" not in st.session_state:
    st.session_state.messages = []

# 지금까지의 대화를 말풍선으로 다시 그리기
for m in st.session_state.messages:
    with st.chat_message(m["role"]):
        st.markdown(m["content"])

# 아래 입력창에 사용자가 뭔가 치면
if prompt := st.chat_input("무엇이든 물어보세요"):
    st.session_state.messages.append({"role": "user", "content": prompt})
    with st.chat_message("user"):
        st.markdown(prompt)

    # AI가 '생성'한 답 받기
    reply = model.generate_content(prompt).text
    st.session_state.messages.append({"role": "assistant", "content": reply})
    with st.chat_message("assistant"):
        st.markdown(reply)

잘 되면, 이제 성격을 바꿔 보거나 맥락을 더 줄 수 있어요:

③ 프롬프트 개선 (예시)
좋아! 여기서 이어서 고쳐줘.
- 사이드바에 '말투 고르기'를 넣어줘(친절한 선생님 / 시크한 전문가 / 귀여운 친구).
  고른 말투가 시스템 프롬프트에 반영되게 해줘.
- '대화 지우기' 버튼을 넣어서 session_state를 비우게 해줘.
- (도전) 유튜브 영상의 댓글 목록을 붙여넣으면, 그 댓글들을 맥락으로 넣어
  '전체 반응을 3줄로 요약해줘'라고 AI에게 시킬 수 있게 해줘.
💡 데이터 API vs AI API

유튜브 API는 ‘사실’을 꺼내 오는 창구(같은 질문엔 같은 답). AI API는 ‘문장’을 생성하는 창구(같은 질문에도 매번 조금씩 다른 답). 둘을 합치면 강력해져요 — 유튜브에서 데이터를 가져와 AI에게 해석시키는 게 오늘의 마지막 도전이에요.

🌱 씨앗 + 재미있는 API 예고

세상엔 키 없이도 쓸 수 있는 재미난 창구가 아주 많아요. 부록에 몇 개 담아 뒀어요. 5일차 나만의 프로젝트나 집에서 자율로 놀 때 꺼내 쓰세요.

  • GBIF — 전 세계 생물 관찰 기록. 어떤 동물이 어디서 발견됐는지 지도에 찍기(2일차 지도와 찰떡).
  • NASA — 오늘 지구 근처를 지나가는 소행성 목록, 오늘의 우주 사진.
  • USGS — 방금 전 세계에서 일어난 지진 실시간 기록.
🌱 내일을 위한 씨앗

오늘 배운 “요청 → 응답(JSON) → 값 꺼내기” 흐름은 세상 거의 모든 API에 그대로 통해요. 마음에 드는 창구 하나를 골라, “이 API로 스트림릿 앱 만들어줘”라고만 하면 됩니다.

🎒 오늘의 체크 & 링크

  • API가 ‘민원창구’라는 걸 이해했다 (요청 → 응답 JSON)
  • 유튜브 채널 통계를 metric 카드로 띄워 봤다
  • 말을 걸면 답하는 나만의 AI 채팅앱을 켜 봤다
  • API 키는 secrets에만 두고, GitHub엔 안 올린다는 걸 안다
