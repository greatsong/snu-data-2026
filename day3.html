1. API란 무엇일까 — 관공서 민원창구

  • API = 관공서 민원창구 (정해진 요청 → 정해진 응답)
  • 정해진 창구(주소)에 요청 → 정해진 양식으로 응답
  • 응답 양식 = 보통 JSON
민원창구API설명
창구 주소요청 URL어디에 물어볼지 정해진 인터넷 주소
제출 서류요청(request)"무엇을 알려줘"라는 우리의 물음
돌려받는 서류응답(response)정해진 양식(JSON)으로 오는 답
신분증API 키"나 이 서비스 쓰는 사람"임을 증명하는 열쇠
🔬 더 알아보기 — API가 대체 뭐예요? — 관공서에서 등본 떼기로 이해하기

API는 ‘정해진 양식을 채워 보내면, 원하는 결과물을 정해진 형식으로 돌려주는 창구’예요. 아래 두 줄을 각각 클릭해 보세요 — 동사무소 등본 떼기와 내 앱의 날씨 받기가 똑같이 ‘요청 → 응답’으로 흐릅니다.

  • 상태코드 = 응답과 함께 오는 번호표 · 셋만 기억
  • 200 성공 · 404 없음(주소 오류) · 429 과다요청(잠깐 쉬기)

JSON — 라벨 붙은 상자

  • JSON = {"라벨": 값} 짝으로 이루어진 상자 · 상자 안에 상자를 넣을 수 있음
  • 오늘 쓸 진짜 창구, KOBIS(영화진흥위원회) 일별 박스오피스 API의 실제 응답으로 봅니다
실제 KOBIS 응답 (일부만 발췌)
{
  "boxOfficeResult": {
    "boxofficeType": "일별 박스오피스",
    "showRange": "20260721~20260721",
    "dailyBoxOfficeList": [
      {
        "rank": "1",
        "movieNm": "영화 제목",
        "openDt": "2026-07-15",
        "audiCnt": "820653",
        "audiAcc": "3410211",
        "scrnCnt": "1943"
      },
      { "rank": "2", "movieNm": "다음 영화", ... }
    ]
  }
}
  • 겉상자 boxOfficeResult 안에 → 목록 상자 dailyBoxOfficeList → 그 안에 영화 한 편 = 상자 하나
  • 값 꺼내기 = 라벨을 따라 들어가기 · "boxOfficeResult 상자에서 dailyBoxOfficeList 목록을 꺼내라"처럼
  • rank 순위 · movieNm 영화명 · audiCnt 그날 관객수 · audiAcc 누적 관객수 · scrnCnt 스크린수

  • "audiCnt": "820653" — 관객수가 숫자가 아니라 따옴표 친 글자로 옴
  • 글자인 채로는 크기 비교·정렬·그래프가 엉망 → 나중에 숫자로 바꾸는 한 단계가 꼭 필요
  • 오늘 오후에 이 함정을 직접 밟고 넘어갑니다 · 미리 기억만

3. 프롬프트 없이 첫 요청 (3단계)

3-1. 브라우저 주소창이 곧 요청이다

  • KOBIS는 키를 주소에 붙여 보내는 방식 → 브라우저 주소창만으로 요청 가능
  • 아래 주소를 붙여넣고, 발급받은키 자리에 내 키를, 날짜는 어제 날짜(예: 20260721처럼 여덟 자리)로 바꿔 엔터
주소창에 붙여넣기 (키·날짜만 바꾸기)
http://www.kobis.or.kr/kobisopenapi/webservice/rest/boxoffice/searchDailyBoxOfficeList.json?key=발급받은키&targetDt=20260721
  • 화면 가득 글자 덩어리(JSON)가 뜨면 성공 · 한 줄로 뭉쳐 보여도 정상
  • 덩어리 속에서 movieNm을 눈으로 찾아보기 → 어제 1위 영화가 정말 있음
  • API 요청 = 특별한 마법이 아니라 정해진 주소를 여는 일이라는 것, 이게 오늘의 첫 발견
⚠️ 키가 틀리면 — 상태코드만 믿지 말 것
  • KOBIS는 키가 틀려도 상태코드는 200 · 대신 응답에 faultInfo(오류 상자)가 옴
  • {"faultInfo": {"message": "유효하지않은 키값입니다"…}}가 보이면 → 키 복사가 잘못된 것
  • 교훈: 번호표(상태코드)와 서류 내용(응답)을 둘 다 확인

import requests
from datetime import datetime, timedelta
from zoneinfo import ZoneInfo

st.title("🎬 박스오피스 원본 JSON 보기")

url = "http://www.kobis.or.kr/kobisopenapi/webservice/rest/boxoffice/searchDailyBoxOfficeList.json"
response = requests.get(url, params={"key": KOBIS_KEY, "targetDt": yesterday})

st.write("상태코드:", response.status_code)

import streamlit as st
import pandas as pd
import requests
from datetime import datetime, timedelta
from zoneinfo import ZoneInfo

st.set_page_config(page_title="박스오피스 대시보드", layout="wide")
st.title("🎬 어제의 박스오피스")

KOBIS_KEY = st.secrets["KOBIS_KEY"]
# 한국 시간 기준 어제 (서버 시계가 외국 기준이어도 정확)
yesterday = (datetime.now(ZoneInfo("Asia/Seoul")) - timedelta(days=1)).strftime("%Y%m%d")

url = "http://www.kobis.or.kr/kobisopenapi/webservice/rest/boxoffice/searchDailyBoxOfficeList.json"
response = requests.get(url, params={"key": KOBIS_KEY, "targetDt": yesterday})

# JSON 상자에서 영화 목록 꺼내기: boxOfficeResult → dailyBoxOfficeList
movies = response.json()["boxOfficeResult"]["dailyBoxOfficeList"]
df = pd.DataFrame(movies)

# 따옴표에 싸여 온 숫자들을 진짜 숫자로 바꾸기 (1절에서 본 함정!)
for col in ["rank", "audiCnt", "audiAcc", "scrnCnt"]:
    df[col] = pd.to_numeric(df[col])

# 1위 영화를 지표 카드 세 장으로
top = df.sort_values("rank").iloc[0]
col1, col2, col3 = st.columns(3)
col1.metric("어제 1위", top["movieNm"])
col2.metric("어제 관객수", f"{top['audiCnt']:,}명")
col3.metric("누적 관객", f"{top['audiAcc']:,}명")

# TOP 10 관객수 막대그래프
st.subheader("📊 TOP 10 관객수")
st.bar_chart(df.set_index("movieNm")["audiCnt"])
  • 코드가 새로 한 일은 셋뿐 · ①목록 꺼내기 ②글자 숫자 → 진짜 숫자 ③카드와 그래프에 놓기
  • 즉 화려한 대시보드의 정체 = JSON에서 값을 꺼내 자리에 놓는 것 · 마법이 아님
✔️ 여기까지 성공 확인
  • 카드 세 장에 어제 1위 영화와 관객수가 차고, 아래에 막대그래프가 뜨면 성공
  • 3절 화면(글자 덩어리)과 지금 화면을 나란히 떠올려 보기 · 같은 데이터입니다
  • 숫자 바꾸는 넉 줄을 지워 보면? → 그래프가 이상해짐 · 궁금하면 나중에 실험

import pandas as pd
import requests
from datetime import datetime, timedelta
from zoneinfo import ZoneInfo

# 기본 설정
st.set_page_config(page_title="KOBIS 일별 박스오피스", layout="wide")
st.title("🎬 어제의 박스오피스 대시보드")

# 한국 시간 기준 어제 날짜 자동 계산 (KOBIS는 YYYYMMDD 형식)
yesterday = datetime.now(ZoneInfo("Asia/Seoul")) - timedelta(days=1)
target_dt = yesterday.strftime("%Y%m%d")
st.caption(f"조회 기준일(어제): {yesterday.strftime('%Y-%m-%d')}")

# 비밀 금고에서 인증키 불러오기
KOBIS_KEY = st.secrets["KOBIS_KEY"]

# API 호출
url = "http://www.kobis.or.kr/kobisopenapi/webservice/rest/boxoffice/searchDailyBoxOfficeList.json"
res = requests.get(url, params={"key": KOBIS_KEY, "targetDt": target_dt})

if res.status_code != 200:
    st.error(f"요청이 실패했어요 (상태코드: {res.status_code})")
    st.stop()

data = res.json()

# 키가 틀리면 faultInfo 상자가 온다
if "faultInfo" in data:
    st.error("인증키가 올바르지 않아요. Secrets의 KOBIS_KEY를 확인해 주세요.")
    st.stop()

box_list = data.get("boxOfficeResult", {}).get("dailyBoxOfficeList", [])
if not box_list:
    st.error("데이터가 없어요. 날짜 형식(YYYYMMDD)을 확인해 주세요.")
    st.stop()

df = pd.DataFrame(box_list)

# 문자열로 오는 숫자들을 진짜 숫자로 변환
for col in ["rank", "audiCnt", "audiAcc", "scrnCnt", "showCnt"]:
    df[col] = pd.to_numeric(df[col])

# 1위 영화 지표 카드
top = df.sort_values("rank").iloc[0]
c1, c2, c3 = st.columns(3)
c1.metric("어제 1위", top["movieNm"])
c2.metric("어제 관객수", f"{top['audiCnt']:,}명")
c3.metric("누적 관객", f"{top['audiAcc']:,}명")

# 보기 좋게 한국어 열 이름으로 정리
표 = df[["rank", "movieNm", "openDt", "audiCnt", "audiAcc", "scrnCnt"]].copy()
표.columns = ["순위", "영화명", "개봉일", "관객수", "누적관객", "스크린수"]
표 = 표.sort_values("순위").reset_index(drop=True)

st.subheader("📋 박스오피스 TOP 10")
st.dataframe(표, use_container_width=True)

# 관객수 상위 5편 막대그래프
st.subheader("📈 관객수 상위 5편")
top5 = 표.sort_values("관객수", ascending=False).head(5)
st.bar_chart(top5.set_index("영화명")["관객수"])

import streamlit as st
import requests

# -----------------------------
# 기본 페이지 설정
# -----------------------------
st.set_page_config(page_title="유튜브 채널 통계 대시보드", page_icon="📊")

st.title("📊 유튜브 채널 통계 대시보드")
st.write("채널 ID를 입력하면 구독자 수, 조회수, 영상 수를 한눈에 보여드려요!")

# -----------------------------
# API 키 불러오기 (secrets 금고에서!)
# -----------------------------
# ⚠️ 절대 코드에 API 키를 직접 쓰지 않아요.
# .streamlit/secrets.toml 파일이나 Streamlit Cloud의 Secrets 설정에
# YOUTUBE_API_KEY = "여기에_실제_키" 형태로 넣어주세요.
API_KEY = st.secrets["YOUTUBE_API_KEY"]

# -----------------------------
# 채널 ID 입력받기
# -----------------------------
channel_id = st.text_input(
    "유튜브 채널 ID를 입력하세요",
    placeholder="예: UC_x5XG1OV2P6uZZ5FSM9Ttw",
)

search_btn = st.button("조회하기 🔍")

# -----------------------------
# 조회 버튼을 눌렀을 때 실행
# -----------------------------
if search_btn:
    if not channel_id.strip():
        st.warning("채널 ID를 먼저 입력해 주세요! 😊")
    else:
        # YouTube Data API v3의 channels 엔드포인트 호출
        url = "https://www.googleapis.com/youtube/v3/channels"
        params = {
            "part": "snippet,statistics",
            "id": channel_id.strip(),
            "key": API_KEY,
        }

        response = requests.get(url, params=params)

        # 상태코드가 200이 아니면 에러 안내
        if response.status_code != 200:
            st.error(
                "😥 API 요청 중 문제가 발생했어요. "
                f"(상태코드: {response.status_code}) "
                "API 키가 올바른지, 요청 한도를 넘지 않았는지 확인해 주세요."
            )
        else:
            data = response.json()
            items = data.get("items", [])

            # 채널을 못 찾은 경우 (items가 비어있음)
            if len(items) == 0:
                st.error(
                    "🔍 채널을 찾을 수 없어요. "
                    "채널 ID가 정확한지 다시 한번 확인해 주세요!"
                )
            else:
                # 필요한 정보 꺼내오기
                channel = items[0]
                channel_name = channel["snippet"]["title"]
                stats = channel["statistics"]

                subscriber_count = int(stats.get("subscriberCount", 0))
                view_count = int(stats.get("viewCount", 0))
                video_count = int(stats.get("videoCount", 0))

                # 영상당 평균 조회수 계산 (0으로 나누기 방지)
                if video_count > 0:
                    avg_view_per_video = round(view_count / video_count)
                else:
                    avg_view_per_video = 0

                st.success(f"'{channel_name}' 채널 정보를 불러왔어요! 🎉")

                # -----------------------------
                # 지표 카드 4개로 보여주기
                # -----------------------------
                col1, col2, col3, col4 = st.columns(4)

                col1.metric("구독자 수", f"{subscriber_count:,}명")
                col2.metric("총 조회수", f"{view_count:,}회")
                col3.metric("영상 수", f"{video_count:,}개")
                col4.metric("영상당 평균 조회수", f"{avg_view_per_video:,}회")

import streamlit as st
from openai import OpenAI

# 페이지 기본 설정
st.set_page_config(page_title="AI 데이터 분석 선생님", page_icon="🤖")
st.title("🤖 AI 데이터 분석 선생님")

# Streamlit Cloud의 secrets 금고에서 API 키를 안전하게 불러오기
client = OpenAI(
    api_key=st.secrets["SOLAR_API_KEY"],
    base_url="https://api.upstage.ai/v1",
)

# 시스템 프롬프트 (AI의 성격 설정)
SYSTEM_PROMPT = "너는 따뜻하고 친절한 데이터 분석 선생님이야. 반드시 순수 한국어로만 답해"

# 세션에 대화 기록이 없으면 처음 한 번만 초기화
if "messages" not in st.session_state:
    st.session_state.messages = [{"role": "system", "content": SYSTEM_PROMPT}]

# 이전 대화 내용을 말풍선으로 화면에 그려주기 (system 메시지는 숨김)
for msg in st.session_state.messages:
    if msg["role"] != "system":
        with st.chat_message(msg["role"]):
            st.markdown(msg["content"])

# 채팅 입력창 (사용자가 메시지를 입력하는 곳)
user_input = st.chat_input("궁금한 것을 물어보세요!")

if user_input:
    # 사용자 메시지를 기록하고 화면에 표시
    st.session_state.messages.append({"role": "user", "content": user_input})
    with st.chat_message("user"):
        st.markdown(user_input)

    # AI 답변 받아오기
    with st.chat_message("assistant"):
        response = client.chat.completions.create(
            model="solar-open2",              # 모델 이름은 그대로 유지
            messages=st.session_state.messages,  # 이전 대화 전체를 함께 전달 (기억 유지)
            reasoning_effort="low",           # 추론 강도 최소 -> 빠른 응답
        )
        answer = response.choices[0].message.content
        st.markdown(answer)

    # AI 답변도 기록에 저장 (다음 질문에 이어서 사용)
    st.session_state.messages.append({"role": "assistant", "content": answer})

import os
import streamlit as st
import requests
from openai import OpenAI
from wordcloud import WordCloud

# 기본 설정
st.set_page_config(page_title="유튜브 댓글 분석", page_icon="💬")
st.title("💬 유튜브 댓글 분석")

YOUTUBE_KEY = st.secrets["YOUTUBE_API_KEY"]

# 한글 폰트 준비 (없으면 한 번만 내려받기)
FONT_URL = "https://raw.githubusercontent.com/google/fonts/main/ofl/nanumgothic/NanumGothic-Regular.ttf"
FONT_PATH = "NanumGothic.ttf"
if not os.path.exists(FONT_PATH):
    with open(FONT_PATH, "wb") as f:
        f.write(requests.get(FONT_URL).content)

video_id = st.text_input("유튜브 영상 ID를 입력하세요", placeholder="예: 9bZkp7q19f0")

if st.button("댓글 가져오기 💬") and video_id.strip():
    url = "https://www.googleapis.com/youtube/v3/commentThreads"
    params = {
        "part": "snippet",
        "videoId": video_id.strip(),
        "maxResults": 100,
        "key": YOUTUBE_KEY,
    }
    res = requests.get(url, params=params)

    if res.status_code != 200:
        st.error(
            "댓글을 가져오지 못했어요 😢 영상 ID가 맞는지, "
            "댓글이 막힌 영상은 아닌지 확인해 주세요. "
            f"(상태코드: {res.status_code})"
        )
    else:
        items = res.json().get("items", [])
        comments = [
            it["snippet"]["topLevelComment"]["snippet"]["textOriginal"]
            for it in items
        ]
        st.session_state.comments = comments  # 세션에 저장해 두기

# 댓글이 준비되면 워드클라우드와 요약 버튼 표시
if "comments" in st.session_state and st.session_state.comments:
    comments = st.session_state.comments
    st.success(f"댓글 {len(comments)}개를 가져왔어요!")

    # 한글 워드클라우드 만들기
    wc = WordCloud(
        font_path=FONT_PATH, width=800, height=400,
        background_color="white",
    ).generate(" ".join(comments))
    st.image(wc.to_array())

    # AI 세 줄 요약
    if st.button("AI 세 줄 요약 🤖"):
        client = OpenAI(
            api_key=st.secrets["SOLAR_API_KEY"],
            base_url="https://api.upstage.ai/v1",
        )
        joined = "\n".join(comments)
        response = client.chat.completions.create(
            model="solar-open2",
            messages=[
                {"role": "system", "content": "너는 여론 분석가야. 반드시 순수 한국어로만 답해."},
                {"role": "user", "content": f"다음 유튜브 댓글들의 전체 반응을 세 줄로 요약해줘.\n\n{joined}"},
            ],
            reasoning_effort="low",
        )
        st.markdown(response.choices[0].message.content)

9. 프로젝트 맛보기 — 서울 혼잡도 실시간 지도 (강사 시연)

  • 이 절은 강사 화면으로 감상하는 구간 · 오늘 배운 것들이 모이면 어디까지 가는지 보여주는 예고편
  • API 한 번 호출 = 딱 지금 이 순간의 스냅샷 (서울 어디가 지금 붐비나)
  • 그 스냅샷을 시각마다 모아 두면 → 시간축이 생김 → 하루 애니메이션
  • 서울 실시간 도시데이터 API(주요 명소 23곳 혼잡도·유동인구)를 매시간 모아 둔 기록을 사용합니다
🔑 핵심 발상 — 스냅샷을 모으면 영상이 된다
  • 사진 한 장(스냅샷) × 여러 시각 = 플립북(애니메이션)
  • 날씨·주가·혼잡도… 실시간 데이터는 모으는 순간 시계열이 됨
  • 5일차 자유 프로젝트에서 "내 주제 + API"를 붙일 때 좋은 본보기
  • 따라 만들기는 선택 (여유 있는 분이 집에서) · 아래 프롬프트는 그때 쓰라고 남겨 둔 것
① 샘플 프롬프트 (집에서 도전용)
스트림릿 앱에 '서울 하루 혼잡 애니메이션 지도'를 만들어줘. 스트림릿 클라우드에 올릴 거야.

- 실시간 혼잡 기록(매시간 모아 둔 CSV):
  https://raw.githubusercontent.com/greatsong/modudata/main/data/seoul_congestion_log.csv
  열: 시각·코드·지역명·혼잡도·인구min·인구max·위도·경도 (서울 명소 23곳)
- '시각' 값은 '2026-06-30 23:00'처럼 분까지만 있는 문자열이야.
  날짜로 바꿀 때 이 형식을 그대로 지정해줘.
- '혼잡도' 값은 숫자가 아니라 여유·보통·약간 붐빔·붐빔 네 가지 말이야.
  그대로 평균을 내면 오류가 나니, 여유=1, 보통=2, 약간 붐빔=3, 붐빔=4 숫자로
  바꾼 뒤에 집계와 색에 써줘.
- '시각'에서 시(hour)를 뽑아, 명소별 시간대 평균 유동인구(인구max)를 구해줘.
- plotly로 지도에 점을 찍고, 시간대별로 넘어가는 애니메이션(슬라이더·재생 버튼)으로 만들어줘.
  점 크기 = 유동인구, 색 = 혼잡도, 배경 타일 없이.
- 재생하면 새벽엔 점이 작고 낮·저녁에 커지게.
- 필요한 라이브러리 목록(requirements.txt)도 같이 줘.
  버전 숫자 없이 이름만, 스트림릿·판다스·넘파이는 기본 설치라 빼고.
  • 여기에 2일차 거주 고령 지도(면)를 배경으로 깔면 → “사는 사람(밤) vs 지금 있는 사람(낮)” 대비 완성 · ③ 개선으로 ↓
③ 프롬프트 개선 (예시)
배경에 서울 행정동 고령비율 단계구분도를 파란 면으로 깔고,
그 위에 혼잡 점을 빨간 버블로 얹어줘(파란 배경과 구분되게, 혼잡한 곳일수록 진하게).
- 행정동 경계(2일차와 같은 파일, 속성은 '코드'·'동'·'구'):
  https://raw.githubusercontent.com/greatsong/modudata/main/data/boundaries/seoul_dong.geojson
- 고령비율은 2일차처럼 인구 데이터에서 계산해줘:
  https://raw.githubusercontent.com/greatsong/modudata/main/data/population_yearly.csv.gz
  이 표의 열은 연도·시도·시군구·동·코드와 나이별 인구 열('계_0세'~'계_100세 이상' 형식)이야.
  '시도'가 '서울특별시'인 2026년 행만 골라, 동마다 '계_65세'부터 '계_100세 이상' 열의 합을
  '계_' 전체 열의 합으로 나눠. 이 표는 한 행이 동 하나라서 따로 묶지 않아도 되고,
  동 이름은 표의 '동' 열에 이미 있어(인구 표에 '구'라는 열은 없으니 쓰지 마).
  경계와는 '코드'로 짝지어(혼잡 데이터의 '코드'가 아니라 인구 표의 '코드', 문자열로).
- 각 명소가 어느 행정동 안에 있는지는 shapely 라이브러리로
  점이 폴리곤 안에 드는지 판정해서 짝지어줘. requirements.txt에 shapely도 추가하고.
- 호버에 지역명·유동인구·혼잡도·그 동네 고령비율이 뜨게.
→ "가산·강남은 밤엔 텅 비고 낮엔 폭발, 고덕·구로는 낮밤이 비슷"이 눈에 보이게.
⏳ 오늘은 ‘감상’이 핵심 (강사 시연 중심)
  • 완성 목표 아님 · 재생 버튼 한 번으로 “실시간이 영상이 된다” 체감
  • 수업 그 시각 실제 붐빔과 대조 = 종이 데이터가 못 주는 감동
  • 주의: 명소 23곳은 통계 아닌 관찰 · 그 순간 값이 빌 수 있음(직전 값으로)

🎒 오늘의 체크 & 링크

  • API 문서에서 볼 세 곳(요청 주소·요청변수·응답 예시)을 안다
  • KOBIS 키를 직접 발급받아 금고(secrets)에만 넣었다
  • 프롬프트 없이 기본 코드로 원본 JSON을 받아 봤다
  • 문서를 붙여넣는 바이브 코딩으로 박스오피스 앱을 완성했다
  • 말을 걸면 답하는 나만의 AI 채팅앱을 켜 봤다
  • 금고 라벨·코드·프롬프트의 이름이 일치해야 한다는 걸 안다
🎡 오늘의 숙제 — 새 창구를 스스로 열어 보기
  • ① 5단계 복습 (도전) — 부록의 키 없는 API 하나를 골라, 오늘 배운 5단계 순서 그대로 앱 만들기 (키가 없으니 2단계는 건너뛰기)
