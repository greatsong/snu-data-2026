1. API란 무엇일까 — 관공서 민원창구

  • API = 관공서 민원창구 (정해진 요청 → 정해진 응답)
  • 정해진 창구(주소)에 요청 → 정해진 양식으로 응답
  • 응답 양식 = 보통 JSON
민원창구API설명
창구 주소요청 URL어디에 물어볼지 정해진 인터넷 주소
제출 서류요청(request)"무엇을 알려줘"라는 우리의 물음
돌려받는 서류응답(response)정해진 양식(JSON)으로 오는 답
신분증API 키"나 이 서비스 쓰는 사람"임을 증명하는 열쇠
  • 상태코드 = 응답과 함께 오는 번호표 · 셋만 기억
  • 200 성공 · 404 없음(주소 오류) · 429 과다요청(잠깐 쉬기)

키 없이 몸풀기 — 고양이 상식과 랜덤 유저

  • 신분증(키) 필요 없는 창구로 연습
  • requests로 요청 → JSON에서 원하는 값 꺼내기
① 샘플 프롬프트
스트림릿(streamlit) 앱 하나(main.py)를 만들어줘. API 첫 연습용이야.

  돌아온 JSON에서 'fact' 값(고양이 상식)을 꺼내 화면에 크게 보여줘.
- 아래에는 https://randomuser.me/api/ 로 요청해서 랜덤 인물 한 명의

- 요청이 성공(상태코드 200)인지 확인하고, 실패하면 친절한 한국어 메시지를 보여줘.
- 초보자용으로 주석을 한국어로 달고, main.py 전체 코드를 한 번에 줘.
  • 누를 때마다 다른 값 → 파일이 아니라 그 순간 창구에서 받아옴 = "살아있는 데이터"
③ 프롬프트 개선 (예시)
좋아! 여기서 이어서 고쳐줘.
- 받아온 JSON 원본도 접었다 펼 수 있게(st.expander) 보여줘. 어떻게 생겼는지 궁금해.
- 고양이 상식을 한국어로 번역해서 함께 보여주면 더 좋겠어.
- 버튼 색과 카드를 따뜻한 노란 톤으로 예쁘게 꾸며줘.
💡 JSON = 라벨 붙은 상자
  • {"이름": "값"} — 라벨과 값이 짝
  • data["fact"] = data 상자에서 fact 라벨 값 꺼내기
  • 어떤 라벨 있는지 → 응답 한번 펼쳐 보면 확인

2. 유튜브 채널 대시보드 만들기

  • 신분증(키) 필요한 진짜 창구 = YouTube Data API v3
  • 받아올 값 → 구독자 수 · 총 조회수 · 영상 수

먼저 열쇠(API 키) 받기 — 3단계

  • 발급처 → 구글 클라우드 콘솔 (console.cloud.google.com)
  • 강사 화면 같이 보며 천천히 · 딱 3걸음
  1. 프로젝트 만들기
    상단 프로젝트 선택 → 새 프로젝트 → 이름 my-youtube → 생성 후 선택
  2. YouTube Data API v3 사용 설정
    API 및 서비스 → 라이브러리 → YouTube Data API v3 검색 → 사용 설정(Enable)
  3. API 키 발급
    API 및 서비스 → 사용자 인증정보 → 사용자 인증정보 만들기 → API 키 → 키 복사
🔒 열쇠, 코드에 그냥 쓰면 안 됨
  • API 키 = 내 신분증
  • 코드에 직접 적어 GitHub 업로드 → 남이 주워 씀
  • 따라서 키는 코드와 따로 보관 (아래 방법)

키를 안전하게 보관하기 — secrets

  • 스트림릿 비밀 금고 → .streamlit/secrets.toml
  • 키는 이 파일에만 · 코드에서는 st.secrets로 꺼내기
.streamlit/secrets.toml
YOUTUBE_API_KEY = "여기에_발급받은_키를_붙여넣기"
🧯 그리고 .gitignore 에 꼭 추가
  • .gitignore에 .streamlit/secrets.toml 한 줄 추가 → "GitHub에 올리지 마"
  • AI에게 ".gitignore에 secrets.toml 넣어줘"라고 부탁

대시보드 만들기

  • 채널 입력 → 구독자·총조회수·영상수 + 영상당 평균 조회수(총조회수 ÷ 영상수)
  • 표시 → st.metric 카드
① 샘플 프롬프트
스트림릿(streamlit) 앱 하나(main.py)를 만들어줘. 유튜브 채널 통계 대시보드야.

- 사용자가 유튜브 채널 ID(예: UC_x5XG1OV2P6uZZ5FSM9Ttw)를 입력하면,
  YouTube Data API v3의 channels 창구에 requests로 요청해서
  구독자 수, 총 조회수, 영상 수를 받아와.
- API 키는 코드에 절대 쓰지 말고 st.secrets["YOUTUBE_API_KEY"] 로 불러와.
- 상단에 st.metric 카드 4개로 보여줘: 구독자, 총 조회수, 영상 수,
  그리고 '영상당 평균 조회수'(총 조회수 ÷ 영상 수, 정수로 반올림).
- 상태코드가 200이 아니거나 채널을 못 찾으면 친절한 한국어 안내를 보여줘.
- 한국어 라벨, 따뜻한 톤, 초보자용 한국어 주석. main.py 전체 코드를 한 번에 줘.
  • 완성된 코드 한 덩어리 도착 · 내 것과 똑같지 않아도 OK, 이런 모습이면 성공
② AI가 준 코드 (예시)
import streamlit as st
import requests

st.title("📺 유튜브 채널 대시보드")

# 🔑 키는 코드에 직접 쓰지 않고, 비밀 금고(secrets)에서 꺼냅니다
API_KEY = st.secrets["YOUTUBE_API_KEY"]

# 채널 ID 입력창 (기본값은 구글 개발자 채널)
channel_id = st.text_input("채널 ID", "UC_x5XG1OV2P6uZZ5FSM9Ttw")

if channel_id:
    # 유튜브 '창구' 주소에 요청을 보냅니다
    url = "https://www.googleapis.com/youtube/v3/channels"
    params = {"part": "statistics", "id": channel_id, "key": API_KEY}
    res = requests.get(url, params=params)

    if res.status_code == 200 and res.json().get("items"):
        # 응답(JSON) 상자에서 statistics 값을 꺼냅니다
        stats = res.json()["items"][0]["statistics"]
        subs = int(stats["subscriberCount"])
        views = int(stats["viewCount"])
        videos = int(stats["videoCount"])
        avg = round(views / videos) if videos else 0

        # metric 카드 4개로 보여주기
        c1, c2, c3, c4 = st.columns(4)
        c1.metric("구독자", f"{subs:,}")
        c2.metric("총 조회수", f"{views:,}")
        c3.metric("영상 수", f"{videos:,}")
        c4.metric("영상당 평균 조회수", f"{avg:,}")
    else:
        st.warning("채널을 찾지 못했어요. 채널 ID를 다시 확인해 주세요. 🙏")
  • 마음에 안 들면 → 대화로 고치기
③ 프롬프트 개선 (예시)
좋아! 여기서 이어서 고쳐줘.
- 채널 이름과 프로필 이미지도 함께 위에 크게 보여줘(snippet 정보 사용).
- 좋아하는 채널 3개를 미리 버튼으로 넣어서, 누르면 바로 통계가 뜨게.
- 영상당 평균 조회수가 100만이 넘으면 카드 옆에 🔥 이모지를 붙여줘.
💡 채널 ID는 어디서?
  • @채널핸들 = 핸들 (채널 ID 아님)
  • 채널 ID = 보통 UC로 시작하는 긴 글자
  • 못 찾으면 → AI에게 "이 핸들의 채널 ID 찾는 법 알려줘"

3. 나만의 AI 채팅앱 만들기

  • 유튜브·고양이 API = 물어보면 정해진 답 (같은 채널 → 늘 같은 통계)
  • AI(LLM) API = 매번 새로운 답을 생성 → 나만의 챗봇

AI 열쇠 받기

  • 예시 → 구글 Gemini
  • 발급처 → Google AI 스튜디오 (aistudio.google.com) → Get API key
  • 모델 → gemini-2.5-flash (빠름·무료 한도 넉넉) · 다른 회사 AI도 방식 동일
🔒 이 키도 똑같이 secrets에
  • AI 키도 신분증 → .streamlit/secrets.toml에만
  • 코드에 직접 쓰지 않기 · GitHub에 올리지 않기 = 오늘의 핵심 습관
.streamlit/secrets.toml (한 파일에 여러 키)
YOUTUBE_API_KEY = "유튜브_키"
GEMINI_API_KEY = "AI_스튜디오에서_받은_키"

대화가 이어지는 채팅앱

  • 부품 3개 → st.chat_input(입력창) · st.chat_message(말풍선) · session_state(대화 기억)
  • 시스템 프롬프트로 성격 부여 ("너는 친절한 데이터 선생님이야")
① 샘플 프롬프트
스트림릿(streamlit)으로 AI 채팅앱 하나(main.py)를 만들어줘.

- 구글 Gemini API(모델 gemini-2.5-flash)를 사용해.
- API 키는 코드에 쓰지 말고 st.secrets["GEMINI_API_KEY"] 로 불러와.
- st.chat_input 으로 사용자가 메시지를 입력하면, AI의 답을
  st.chat_message 말풍선으로 보여줘.
- session_state에 대화 기록을 저장해서, 이전 대화를 기억하며 이어서 답하게 해줘.
- 시스템 프롬프트로 '너는 따뜻하고 친절한 데이터 분석 선생님이야'라는 성격을 줘.
- 한국어로 대화하고, 초보자용 한국어 주석을 달아. main.py 전체 코드를 한 번에 줘.
  • 아래는 받게 될 코드의 모습 예시 · 라이브러리 이름·형태 조금 달라도 OK
② AI가 준 코드 (예시)
import streamlit as st
import google.generativeai as genai

st.title("🤖 나만의 AI 채팅")

# 🔑 AI 키도 비밀 금고에서 꺼냅니다
genai.configure(api_key=st.secrets["GEMINI_API_KEY"])

# 성격 부여(시스템 프롬프트) + 매번 새 답을 '생성'하는 모델
model = genai.GenerativeModel(
    "gemini-2.5-flash",
    system_instruction="너는 따뜻하고 친절한 데이터 분석 선생님이야."
)

# 지난 대화를 기억하는 메모장
if "messages" not in st.session_state:
    st.session_state.messages = []

# 지금까지의 대화를 말풍선으로 다시 그리기
for m in st.session_state.messages:
    with st.chat_message(m["role"]):
        st.markdown(m["content"])

# 아래 입력창에 사용자가 뭔가 치면
if prompt := st.chat_input("무엇이든 물어보세요"):
    st.session_state.messages.append({"role": "user", "content": prompt})
    with st.chat_message("user"):
        st.markdown(prompt)

    # AI가 '생성'한 답 받기
    reply = model.generate_content(prompt).text
    st.session_state.messages.append({"role": "assistant", "content": reply})
    with st.chat_message("assistant"):
        st.markdown(reply)
  • 잘 되면 → 성격 바꾸기 · 맥락 더 주기
③ 프롬프트 개선 (예시)
좋아! 여기서 이어서 고쳐줘.
- 사이드바에 '말투 고르기'를 넣어줘(친절한 선생님 / 시크한 전문가 / 귀여운 친구).
  고른 말투가 시스템 프롬프트에 반영되게 해줘.
- '대화 지우기' 버튼을 넣어서 session_state를 비우게 해줘.
- (도전) 유튜브 영상의 댓글 목록을 붙여넣으면, 그 댓글들을 맥락으로 넣어
  '전체 반응을 3줄로 요약해줘'라고 AI에게 시킬 수 있게 해줘.
💡 데이터 API vs AI API
  • 유튜브 API = 사실을 꺼내는 창구 (같은 질문 → 같은 답)
  • AI API = 문장을 생성하는 창구 (같은 질문 → 매번 조금 다름)
  • 둘을 합치면 강력 → 유튜브 데이터 가져와 AI에게 해석시키기 = 오늘의 마지막 도전

🌱 씨앗 + 재미있는 API 예고

  • 키 없이 쓰는 재미난 창구 많음 · 부록 참고 · 5일차 프로젝트/자율놀이에 활용
  • GBIF — 전 세계 생물 관찰 기록 → 지도에 찍기(2일차 지도와 찰떡)
  • NASA — 오늘 지구 근처 지나는 소행성 목록·오늘의 우주 사진
  • USGS — 실시간 전 세계 지진 기록
🌱 내일을 위한 씨앗
  • "요청 → 응답(JSON) → 값 꺼내기" 흐름 = 거의 모든 API에 통용
  • 창구 하나 골라 → "이 API로 스트림릿 앱 만들어줘"

🎒 오늘의 체크 & 링크

  • API가 '민원창구'라는 걸 이해했다 (요청 → 응답 JSON)
  • 유튜브 채널 통계를 metric 카드로 띄워 봤다
  • 말을 걸면 답하는 나만의 AI 채팅앱을 켜 봤다
  • API 키는 secrets에만 두고, GitHub엔 안 올린다는 걸 안다
